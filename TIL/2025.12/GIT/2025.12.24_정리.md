# GIT 이란?
- Git - "분산"형 버전 관리 시스템 (동시성제어 & 충돌제어)
- 작업공간(Working Directory) / 준비영역(Staging Area) / 저장소(Repository) 3가지로 구성
  1) git init : 저장소 초기화
  2) git add : 로컬 -> 준비영역
  3) git commit : 준비영역 -> 저장소

# Branch?
- 브랜치 : 커밋을 가리키는 포인터(책갈피 느낌)
  1) git branch : 브랜치 추가 (=git switch -c)
  2) git switch : 브랜치 이동
     
## HEAD?
 지금 내가 어디있는지. HEAD는 무조건 한개. "현재 브랜치를 추적"

## 브랜치 전략? 
 - 이름 규칙 : git flow 또는 github flow 등이 있다.
 - main 혹은 master : 배포 가능한 안정 버전
  > 그냥 짜증나니까 git init 하자마자 "git branch -M main" 명령어 바로 때려서 바꿔주자.
 - develop : 개발 통합 브랜치
 - feature : 새 기능 개발. (ex : feature/account)
 - bugfix : 버그 수정. (ex : bugfix/login-error)
 - hotfix : 긴급 수정. (ex : hotfix/vulnerability)
  
## 브랜치 머징?
 - 각자 작업한걸 모아 모아 완성본 만들기
   1) Fast-Forward : main을 feature의 최종적으로 커밋한 버전으로 옮겨버린다.
   2) 3-way-Merge : 작업자가 두명 이상일때. 합치고 합침.
      - 결국, git merge 명령어만 사용하면 되고, 방식은 Git에서 알아서한다.

## Git Remote?
 1) git push : 로컬 레포지토리에서 원격 레포지토리로 업데이트
 2) git pull : 원격 레포지토리에서 로컬 레포지토리로 다운로드
 3) git remote : 인터넷 or 네트워크에 연결된 원격 저장소에 접속 & 연결하는 명령어
     - 그전에, git remote add 를 때려서 원격 저장소 추가해야함.
     - git remote -v : 원격 저장소 확인
