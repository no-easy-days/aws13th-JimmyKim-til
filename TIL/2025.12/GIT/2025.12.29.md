### 변수와 자료형 모르는 부분

#### 리스트 자료형 ---- 대괄호
list.append("뭐시기") : 리스트 끝에 값을 붙이기
list.insert(1, "뭐시기") : 해당 순서에 값을 끼워넣음
len(list) : 리스트에 요소 값 갯수 반환
list.remove("뭐시기") : 해당 요소 삭제
list.pop() : 마지막 요소를 꺼낸다. (큐 개념으로 접근하면 활용도가 높다)

#### 딕셔너리 자료형 ---- 중괄호 + 콜론
dict["뭐시기"] = "저시기" : 새로운 키&밸류 값 추가 또는 수정
dict.update({"뭐시기": "저시기", "거시기": "그시기"}) : 새로운 키&밸류 값을 "여러개" 추가
dict.get("키") : 밸류 값을 반환
!!! 없는 키를 호출 하면, None / 대괄호로 호출하면, KeyError 발생
del dict["키"] : 해당하는 키 & 밸류 값 삭제
dict.clear() : 전부 삭제

dict.keys() : 키값 모두 가져오기 
dict.values() : 밸류 값 모두 가져오기 
dict.items() : 모든 키&밸류 쌍 가져오기 (if 문에서 활용 가능) --> for key, value in dict.items(): 형태

#### 튜플 자료형 ---- 소괄호
UD가 불가능하다. C는 처음에만, R은 가능
언패킹 : 튜플의 밸류 값을 여러 변수에 한번에 담을 수 있다. 순서&인덱싱을 가지니까.

#### 세트 자료형 ---- 중괄호
"순서&인덱싱 없음", "중복 허용 불가 " > 리스트를 세트 형태로 변환하여 중복 제거 가능
set.add() : 요소값 추가. (순서가 없기 때문에 append를 쓸 수 없음)
set.update([]) : 요소값 여러개 한번에 추가. (대괄호 넣어야함!)
set.discard("뭐시기") : 요소값 삭제, 요소값이 없어도 에러 나지 않음.
 
### for 문
for i in range(): 범위로 루프돌리기
for i in enumerate("뭐시기", start=1): 순번과 값을 동시에 받을 수 있음. 예시에서는 1부터. (자바 enum과 동일 개념)
for i in zip(listA, listB, listC): 리스트를 짝지어서 루프 돌리기. 예시에서는 A,B,C 리스트 3개를 지퍼처럼 짝지음

### Pycharm 디버깅 도구 활용법
1) 특정 지점에 BreakPoint를 지정 후 벌레 모양 클릭
2) 그러면 디버그 창이 활성화되고, "seperate watch"를 활성화 하면 "실시간 변화값 추적" 가능.
3) 한 단계 넘기고 싶으면, "Step Over(F8)" / 함수 안쪽으로 들어가고 싶으면 "Step Into(F7)" / 함수 밖으로 나가려면 "Step Out(F9)"
4) 키맵에서 toggle line breakpoint --> ctrl + shift + B 로 지정해서, 단축기를 활용하자.
5) BreakPoint 오른쪽 마우스 클릭 : "조건 부여해서 그 순간 멈출 수 있도록" 설정 (For문 또는 변수값 할당 등등)

### 함수와 모듈
수학의 y = f(x)와 동일하다. x는 입력값이고 f(x)는 결과값(출력값)이 된다.
return이 없다면 None을 반환한다. (NULL값 반환)
 - """독스트링 : 함수의 설명 섹션, 함수 이름 다음 바로 첫 줄에 작성하도록 권장한다. """

#### 파라미터 vs 아규먼트
 - 파라미터(매개변수) : 함수 "정의" 시 사용하는 변수 이름
 - 아규먼트(인자) : 함수 "호출" 시 실제로 전달해야하는 값

#### 기본값 매개변수(Default Parameter)
 - def function(a, b=10): 과 같이 선언하고, "b에 인자값을 받지 않으면 10을 기본값으로" 들고간다.
 - 반드시 뒷쪽에 배치해야한다.

#### 키워드 인자(Keyword Argument)
 - 요 자리는 이게 들어가야한다! 라는 것을 알려주는 역할.
 - 만약, "이름 / 나이 / 도시" 로 매개변수가 정의 되어있다 할지라도, 키워드 인자를 쓰면 "나이 / 이름 / 도시" 형태도 가능(순서 무관).

#### 변수의 범위(Scope)
 - "LEGB 규칙" : Local(함수 내부) -> Enclosing(외부 함수) -> Global(모듈 전역) -> Built-in(내장 함수) 의 순서로 변수 값을 탐색한다.
 - global 키워드 : 특정 함수 내부에서 "전역 변수"를 수정할 때 사용할 수 있지만, 쓰지 마라!!


### 클래스 기초
 - 용어 정리
   1) 클래스 : 객체를 만들기 위한 가장 큰 틀의 설계도.
   2) 인스턴스 : 설계도를 바탕으로 만든 실제 객체. 오브젝트라고 부르는 곳도 있음
   3) 속성 : 객체가 가진 정보.
   4) 메소드 : 객체가 할 수 있는 동작
   5) 매직 메소드 : 클래스 호출하여 인스턴스 생성 시 "제일 처음 자동으로 호출"하는 특별한 메소드

 - self 란? : 클래스로 선언된 "나 자신"을 가리키는 키워드, (자바에서 this와 같은 역할)
                메소드 안에서 인스턴스의 속성에 접근할 때 사용.
                "생성자 매직 메소드(__init__) 에서는 반드시 self를 써야한다."
 - 클래스는 한가지 일만 하는게 좋다.

### 모듈과 패키지
 - 모듈 : 하나의 .py 파일 / 패키지 : 모듈을 폴더로 묶은 것 / 라이브러리 : 패키지들을 묶은 것

     2) pip install -r requirements.txt : 텍스트 파일에 있는 패키지 리스트를 내 PC에 한번에 설치

### 예외 처리 (Exception Handling)
일종의 "안전망", 에러가 발생하는 것을 어떻게 대처할지 프로세스를 구현해둔 것.
 - 구조
   try: 예외 발생할 수 있는 코드
   except: 특정 예외 처리
   else: 예외가 발생하지 않았을 때만 실행
   finally: 예외 발생 여부에 관계없이 항상 실행

 - raise : 의도적으로 예외를 발생시킨다.
            raise가 발생했다면 그걸 받아내는 exception 처리 구문도 필요하다.
