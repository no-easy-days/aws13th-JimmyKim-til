### Custom AMI
- 내 입맛에 맞게 구성한 서버환경을 이미지화 (디스크)
- 온디맨드(On-demand, 수요가 발생하는 그 즉시)로 서버 생성이 가능하며 동일한 환경을 항상 ***"재현"*** 가능. (= Human-Error 방지)
  > Scale을 유동적으로 변경 가능하다. → Auto-scaling


- 뭔가 설치했던 Daemon들이 있으면, 이미지를 불러서 부팅했을 때도 켜져야하기 때문에 enable 해줘야한다.
- 생성할 EC2 선택 → Actions → Image and Templates → Create Image 해서 생성한다. 


### ELB(Elastic Load Balancer)
- 부하 분산기
  <img width="1072" height="621" alt="image" src="https://github.com/user-attachments/assets/7fedf451-319a-4608-8dba-adb6083eac16" />


  (1) 트래픽 분산 : 순차접속 ~ "Round-Robin", 때에 따라서는 Health-Check를 활용해서 대상이 멀쩡한지 확인해준다.


  (2) 하나의 Endpoint를 만들 수 있다.


- EC2 메뉴 밑에 Load Balancer Tab이 있다.
- ***내가 연결시켜줄 내부의 리소스들과는 별개의 Security Group을 가진다!!!*** 주의하자.
- Load Balancer는 Application Load Balancer(ALB) / Network Load Balancer(NLB) 둘 중에 골라서 쓰면 된다.


  (1) ALB : 7 Layer 기반. 서비스(특정 프로토콜) 기준으로 부하 분산해준다. REST api 기반 접속일 때 적용하면 좋다.


  (2) NLB : 4 Layer 기반. Port 번호 기준으로 부하 분산해준다. ***★TCP!!★, HTTP는 불가능함!!! 4계층이니까***

  
- ELB를 어느 위치에 놓을지 정할 수 있다.


  (1) Internet-Facing : 외부(Public) IP Address ~ 내부(Private) IP Address를 접점으로 한다.
     #### PUBLIC SUBNET에 생성해야한다!!


  (2) Internal : 내부(Private), 그러니까 특정 VPC 내부의 Private IP Address를 접점으로 한다.
     #### PRIVATE SUBNET에 생성해야한다!!
