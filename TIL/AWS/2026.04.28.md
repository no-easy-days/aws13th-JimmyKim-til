# EKS 배포 명령어 정리

yaml 파일로 처리되지 않는 kubectl / eksctl / AWS CLI 명령어 모음.

---

## 0. 사전 준비 - 툴 설치 (Ubuntu 기준)

```bash
# AWS CLI
apt-get -y install unzip
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip && ./aws/install
aws configure   # Access Key, Secret Key, Region, Format 입력

# eksctl
curl -fsSL "https://github.com/weaveworks/eksctl/releases/latest/download/eksctl_$(uname -s)_amd64.tar.gz" \
| tar xz -C /usr/local/bin

# kubectl - 공식 문서 참조
# https://docs.aws.amazon.com/ko_kr/eks/latest/userguide/install-kubectl.html

# Helm
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# 자동완성 세팅
echo 'complete -C "$(which aws_completer)" aws' >> ~/.bashrc
eksctl completion bash | sudo tee /etc/bash_completion.d/eksctl > /dev/null
echo "source <(kubectl completion bash)" >> ~/.bashrc
source ~/.bashrc
```

## 1. 변수 설정

```bash
export CLUSTER_NAME=<클러스터이름>
export REGION=ap-northeast-2
export ACCOUNT_ID=<AWS 계정 ID>   # aws sts get-caller-identity 명령으로 확인
export VPC_ID=<VPC ID>   # AWS Console - VPC에서 확인
```

---

## 2. EKS 클러스터 생성

```bash
eksctl create cluster \
  --vpc-public-subnets <퍼블릭서브넷1>,<퍼블릭서브넷2> \
  --name $CLUSTER_NAME \
  --region $REGION \
  --version <쿠버네티스버전> \
  --nodegroup-name <노드그룹이름> \
  --node-type <인스턴스유형> \
  --nodes <원하는노드수> \
  --nodes-min <최소노드수> \
  --nodes-max <최대노드수>
```

생성 후 kubeconfig 업데이트:
```bash
aws eks update-kubeconfig --region $REGION --name $CLUSTER_NAME
```

> 클러스터 생성 시 CloudFormation 스택이 자동 생성됨. 에러 발생 시 CloudFormation에서 스택 삭제 후 재시도.

## 4. ECR 이미지 Push

```bash
# ECR 로그인
aws ecr get-login-password --region $REGION | \
  docker login --username AWS --password-stdin $ACCOUNT_ID.dkr.ecr.$REGION.amazonaws.com

# 이미지 빌드 & 태그 & Push
docker build -t board-app:1 ./board-app
docker tag board-app:1 $ACCOUNT_ID.dkr.ecr.$REGION.amazonaws.com/board-app:1
docker push $ACCOUNT_ID.dkr.ecr.$REGION.amazonaws.com/board-app:1
```

## 5. AWS Load Balancer Controller 설치 (ALB Ingress 사용 전제)

```bash
# IAM Policy 생성 (최초 1회)
curl -O https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/v2.14.1/docs/install/iam_policy.json

aws iam create-policy \
  --policy-name AWSLoadBalancerControllerIAMPolicy \
  --policy-document file://iam_policy.json

# IRSA 생성
eksctl create iamserviceaccount \
  --cluster $CLUSTER_NAME \
  --namespace kube-system \
  --name aws-load-balancer-controller \
  --role-name AmazonEKSLoadBalancerControllerRole \
  --attach-policy-arn arn:aws:iam::$ACCOUNT_ID:policy/AWSLoadBalancerControllerIAMPolicy \
  --approve \
  --override-existing-serviceaccounts

# Helm으로 설치
helm repo add eks https://aws.github.io/eks-charts
helm repo update
helm install aws-load-balancer-controller eks/aws-load-balancer-controller \
  -n kube-system \
  --set clusterName=$CLUSTER_NAME \
  --set serviceAccount.create=false \
  --set serviceAccount.name=aws-load-balancer-controller \
  --set image.repository=602401143452.dkr.ecr.ap-northeast-2.amazonaws.com/amazon/aws-load-balancer-controller \
  --set region=$REGION \
  --set vpcId=$VPC_ID
```

> 설치 확인: `kubectl describe pod -n kube-system`

## 6. IRSA - ServiceAccount 생성 (Secrets Manager 접근용)

Deployment에서 `secret-sa` ServiceAccount를 사용하므로 반드시 네임스페이스 지정 필요.

```bash
eksctl create iamserviceaccount \
  --name secret-sa \
  --namespace board-app-eks \
  --region $REGION \
  --cluster $CLUSTER_NAME \
  --attach-policy-arn arn:aws:iam::aws:policy/SecretsManagerReadWrite \
  --approve \
  --override-existing-serviceaccounts
```

> 주의: `--namespace` 빠지면 default 네임스페이스에 생성되어 Deployment가 SA를 못 찾음

Secret CSI 드라이버 설치:
```bash
# Helm repo 추가
helm repo add secrets-store-csi-driver https://kubernetes-sigs.github.io/secrets-store-csi-driver/charts
helm repo update

# 설치 (시크릿 동기화 옵션 포함)
helm install -n kube-system csi-secrets-store \
  --set syncSecret.enabled=true \
  --set enableSecretRotation=true \
  --set rotationPollInterval=15s \
  secrets-store-csi-driver/secrets-store-csi-driver

# CSI 드라이버에 토큰값 추가
kubectl edit csidriver secrets-store.csi.k8s.io
# 아래 내용 추가:
# tokenRequests:
#   - audience: sts.amazonaws.com

# ASCP 설치
kubectl apply -f https://raw.githubusercontent.com/aws/secrets-store-csi-driver-provider-aws/main/deployment/aws-provider-installer.yaml
```

## 7. EKS Console 접근 권한 (IAM 유저 → K8s RBAC 매핑)

```bash
eksctl create iamidentitymapping \
  --cluster $CLUSTER_NAME \
  --region $REGION \
  --arn arn:aws:iam::$ACCOUNT_ID:user/<IAM유저이름> \
  --username <IAM유저이름> \
  --group system:masters
```


## 10. 트러블슈팅용 임시 mysql client Pod

RDS에 직접 접속해서 DB/유저 확인할 때 사용.

```bash
kubectl run -it --rm mysql-client \
  --image=mysql:8.0 \
  --restart=Never \
  -n board-app-eks \
  -- mysql -h '<ENDPOINT주소>' \
           -u admin -p'<RDS마스터패스워드>'
```

접속 후 확인 쿼리:
```sql
SHOW DATABASES;
SELECT user, host FROM mysql.user WHERE user = 'boarduser';
SHOW GRANTS FOR 'boarduser'@'%';
```
