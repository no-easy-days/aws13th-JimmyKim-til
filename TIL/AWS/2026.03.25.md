## (1) Referencing Security Group (참조)
<img width="1117" height="690" alt="image" src="https://github.com/user-attachments/assets/4b3858d8-0a8e-4dac-a7cb-1baa95d75378" />


- Security Group 생성 시, ***"Source"에 다른 Security Group을 적용***할 수 있는데 이를 Referencing 했다고 한다.

  
- 만약 Source에 특정 Security Group을 적용했다면, ***그 Security Group이 적용된 인스턴스로 부터 오는 트래픽에 대해 Rule을 설정***하는 것이다!

  
- 결국 이것도 Identity 기반이기 때문에, 참조된 Security Group의 Inbound ~ Outbound Rule은 사실 중요하지 않다.
  > 아키텍팅하다가 헷갈리면 Security Group 타고 타고 들어가보자.
