### telegraf, influxDB, grafana를 활용한 모니터링 에이전트 구현
<img width="857" height="431" alt="image" src="https://github.com/user-attachments/assets/a7dcd62d-3a43-4848-8be8-8db2bcb632a8" />


1. telegraf : 어떤 서버의, 어떤 리소스 사용량을 얼마의 간격으로 수집할지 설정할 수 있는 일종의 ***리소스 모니터링 "에이전트"***
   > 모니터링 대상 서버 마다 설치되어 있어야 한다. 정보를 수집해야하니까. 뿌려주는건 커스터마이징 가능.


2. influxDB : ***시계열(time-serial) 데이터베이스.*** 일정 시간 간격으로 데이터를 수집해서 저장한다.
3. grafana : 저장되어있는 데이터(DB가 필요하긴하다. Data Source에서 지정 가능)를 바탕으로 그래프, 대시보드 및 수치화 Tool


#### (1) telegraf 에이전트 설치하기

1. telegraf 설치를 위한 레포지토리 설정 (설정 완료되면 패키지 다운로드)
   ```
   cat <<EOF | tee /etc/yum.repos.d/influxdb.repo
   [influxdb]
   name = InfluxDB Repository - RHEL \$releasever
   baseurl = https://repos.influxdata.com/rhel/\$releasever/\$basearch/stable
   enabled = 1
   gpgcheck = 0
   gpgkey = https://repos.influxdata.com/influxdb.key
   EOF

   ※ influxDB랑 벤더사가 동일하다. 그래서 레포지토리도 같은걸 깔면 된다.
   ```
2. telegraf 컨피그 파일 수정
   - vi /etc/telegraf/telegraf.conf
   ```
   [global_tags]
   # Configuration for telegraf agent
   [agent]
        interval = "10s"
        debug = false
        hostname = "server-hostname"
        round_interval = true
        flush_interval = "10s"
        flush_jitter = "0s"
        collection_jitter = "0s"
        metric_batch_size = 1000
        metric_buffer_limit = 10000
        quiet = false
        logfile = ""
        omit_hostname = false

    ###############################################################################
    #                                  OUTPUTS                                    #
    ###############################################################################

    [[outputs.influxdb]]
        urls = ["http://localhost:8086"] # InfluxDB가 설치된 서버의 IP를
        database = "telegraf" # 데이터베이스 이름, 생성이 되어있지 않으면 자동 생성됨
        timeout = "10s"
        username = "admin" # InfluXDB 기본 계정
        password = "admin"
        retention_policy = ""

    ###############################################################################
    #                                  INPUTS                                     #
    ###############################################################################

    [[inputs.cpu]]
        percpu = true
        totalcpu = true
        collect_cpu_time = false
        report_active = false
    [[inputs.disk]]
        ignore_fs = ["tmpfs", "devtmpfs", "devfs"]
    [[inputs.diskio]]
    [[inputs.mem]]
    [[inputs.net]]
    [[inputs.system]]
    [[inputs.swap]]
    [[inputs.netstat]]
    [[inputs.processes]]
    [[inputs.kernel]]
   ```


#### (2) influxDB 구축하기


1. telegraf 설치를 위한 레포지토리 설정 (설정 완료되면 ① 패키지 다운로드 → ② 데몬 활성화하면 된다.)
 ※ 위에서 적은거 처럼 influxDB랑 벤더사가 동일하다. 그래서 레포지토리도 같은걸 깔면 된다.
 
2. Influx 접속해서 telegraf 연동 확인
   - 명령어 : influx
   - 기본 포트 : 8086


#### (3) grafana 구축하기

1. grafana 설치를 위한 레포지토리 설정 (설정 완료되면 ① 패키지 다운로드 → ② 데몬 활성화하면 된다.)
   ```
   cat <<EOF | tee /etc/yum.repos.d/grafana.repo
   [grafana]
   name=grafana
   baseurl=https://packages.grafana.com/oss/rpm
   repo_gpgcheck=1
   enabled=1
   gpgcheck=0
   gpgkey=https://packages.grafana.com/gpg.key
   sslverify=1
   sslcacert=/etc/pki/tls/certs/ca-bundle.crt
   EOF
   ```

2. grafana 접속해서 연동 세팅 및 확인
   - 기본 포트 : 3000
   <img width="1110" height="486" alt="image" src="https://github.com/user-attachments/assets/eb0fb3e9-dc25-49cb-ab8e-8b4b14c2bd0e" />


   - 환경 변수 설정
     - 기본값으로 대시보드상에 ${DS_INFLUXDB_TELEGRAF} 를 통해 datasource로 접근가능하도록 환경변수가 구성된 상태이지만, 실제 환경변수는 존재X.
     - 따라서 대시보드의 설정에서 환경변수를 추가해줘야한다.


### prometheus, node-exporter, grafana를 활용한 모니터링 에이전트 구현

1. node-exporter : telegraf의 역할을 수행. 리소스 모니터링 데이터 수집 에이전트 프로그램.
2. prometheus : InfluxDB의 역할을 수행. 시계열로 데이터를 저장하긴 하지만 DB라고 하기엔 애매한 프로그램.
3. grafana : 저장되어있는 데이터(DB가 필요하긴하다. Data Source에서 지정 가능)를 바탕으로 그래프, 대시보드 및 수치화 Tool
   > 일반적으로 prometheus와 node-exporter는 거의 하나의 프로그램처럼 여긴다.


#### (1) node-exporter 에이전트 설치

1. node-exporter 설치 및 압축 해제
   - wget https://github.com/prometheus/node_exporter/releases/download/v1.7.0/node_exporter-1.7.0.linux-amd64.tar.gz
   - tar -xvf node_exporter-1.7.0.linux-amd64.tar.gz
     > tar 명령어 ~ x : eXtract, 압축해제 / v : View, 과정 보기 / f : File, 파일 지정하기 / C : Compress : 압축


2. node-exporter 실행
   - ./node-exporter
   - 데몬 형태로 등록해서 실행도 가능하다. ***사용 포트 : 9100***
   - 끄고 싶으면 PID 찾아서 프로세스 킬 하면 된다.

  
#### (2) prometheus 설치

1. prometheus 설치 및 압축 해제
   - wget https://github.com/prometheus/prometheus/releases/download/v2.45.0/prometheus-2.45.0.linux-amd64.tar.gz
   - tar -xvf prometheus-2.45.0.linux-amd64.tar.gz

2. prometheus 컨피그 파일 수정
   - vi prometheus.yml
     ```
     - job_name: "prometheus"
     # metrics_path defaults to '/metrics'
     # scheme defaults to 'http'.
       static_configs:
        - targets: ["localhost:9090"]
     # self health-check

     - job_name: "node_exporter"
       static_configs:
        - targets: ["localhost:9100"]

     ※ JSON 형태로 이루어짐
     ```
3. prometheus 실행
   - ./prometheus
   - 웹 페이지를 띄우기 때문에, 브라우저 통해서 확인 가능하다. ***사용 포트 : 9090***

> Grafana는 위와 동일하다. 구축 완료 후, Data Source를 Prometheus로 지정하고, Dashboard도 node-exporter 호환 템플릿 적용하면됨.
