### NFS 서버
- Network File System. 원격지에 존재하는 파일을 마치 내 로컬 파일인 것 처럼 사용 가능하다.
- 서버~클라이언트 모델, 데이터 송&수신 시 랜덤하게 포트가 부여된다. (기본 포트는 TCP : 2049, RPC : 111)

#### 서버 측 구성
1. NFS 구성 패키지 설치
   - yum install -y nfs-utils
2. 공유 대상 디렉토리 생성 및 권한 부여
   - chmod -R 777 적절하게 부여하면 됨. (PRISMAproduction 구축할 때 매번 하던 짓이네)
3. NFS 구성 파일 수정
   - vi /etc/exports ← ***nfs-utils를 설치하면, 이 구성 파일이 생긴다.***
     ```
     /shared          *(rw)
       ↑              ↑
     <공유대상폴더>   <폴더 내 공유할 파일>(권한)
     ```
4.  NFS 서비스 재시작 및 활성화
   - systemctl restart nfs-server & systemctl enable nfs-server
6.  NFS 구성 결과 확인
   - showmount -e <서버주소>


#### 클라이언트 측 구성
1. 클라이언트 측 NFS 구성 패키지 설치 및 마운트 포인트가 될 폴더 생성 & 권한 부여 (동일)
2. NFS 서버 연동
   - mount -t nfs <서버주소>:<공유대상폴더> <마운트포인트가 될 폴더>
3. 설정 저장 (서버 부팅 시 자동으로 마운트 처리)
   - vi /etc/fstab
     ```
     192.168.0.100:/shared        /my_folder                nfs        default              0  0
         ↑           ↑              ↑                        ↑           ↑                   ↑
     <서버주소>:<공유대상폴더>    <마운트포인트가 될 폴더>    <Type>    <파일 시스템 옵션>    <덤프 설정 등...>
     ```
