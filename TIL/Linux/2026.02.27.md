### Ubuntu 리눅스 구축 Tip
- 기본 레포지토리가 유효하기 때문에 별도 변경 필요는 없다. 그렇지만 패키지 매니저를 업데이트 해주는 것이 충돌을 방지한다.
  > apt update -y (항상 패키지 설치 전에 한번 해주자)
- Ubuntu용 vi 편집기가 버전이 너무 낮아서 메롱이다. vim 패키지를 깔아서 사용하자.
  > apt install -y vim
- 보통 Ubuntu는 패키지를 설치하면 자동으로 동작까지 실행시킨다.
- Ubuntu 자체 방화벽(UFW, Ubuntu FireWall)이 존재하지만, 기본적으로 비활성화 상태이다.
- CentOS 계열과 달리, sshd가 기본 내장되어 있지 않기 때문에 별도 설치가 필요하다.
- 첫 설치 이후, 별도의 root 계정 PW 등록하는 부분이 없기 때문에 명령어를 통해 등록해주자.
  > sudo passwd root (user1로 로그인 하여 switch user 프로세스로 등록)

### Ubuntu(Linux Mint) 기준 sshd 구축
1. SSH 서버 패키지 설치
   > apt install -y openssh-server
2. SSH 데몬 Config 변경
   <img width="1136" height="401" alt="image" src="https://github.com/user-attachments/assets/f0fa6b2d-c39c-4341-930f-3e93ab06fdcd" />


   > vi /etc/ssh/sshd_config ("PermitRootLogin=yes" 로 변경. Default는 root의 접속을 허용하지 않는다.)
   

3. 서비스 재시작
   > systemctl restart sshd


※ 클라이언트는 서버로부터 Default 값으로 ***RSA 방식의 공개 키***를 받는다. 실제 키가 같은 것도 확인 가능.


### Linux 서버 방화벽 설정하기
- 흔히 말하는 "Well-Known" 포트는 암묵적인 약속이다.
- 반드시 그 포트(L4, 전송 계층)로 서비스(L7, 응용 계층)를 제공해야하는 것을 뜻하지 않는다.
- 포트는 일종의 통로 역할이며, 충분히 변경할 수 있다.


1. 방화벽 활성화
   > systemctl restart firewalld


2. 기존 방화벽 설정 확인
   > firewall-cmd --list-all


  <img width="681" height="336" alt="image" src="https://github.com/user-attachments/assets/090db3f7-51c6-428d-a69f-ae995ed6cdde" />

  ※ ***서비스 자체를 허용하면, 그 서비스가 연관된(Related) 포트는 자동으로 허용***되도록 설계되었다.
  ※ 커스텀 포트를 활용하고 싶다면, 방화벽은 그 정보를 모르기 떄문에 포트 허용이 필수적이다!


3. 방화벽 설정 명령어
  - firewall-cmd --permanent --add-port=80/tcp (포트 허용 문법)
  - firewall-cmd --permanent --add-service=http (서비스 허용 문법)


4. 설정한 정책 반영
   > firewall-cmd --reload
