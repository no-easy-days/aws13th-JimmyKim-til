### RAID (Redundant Array Independant Disk)
- 다수의 디스크를 배열(Array)을 통해 ***용량을 늘리거나 or 속도를 늘리거나 or 안정성을 늘리거나*** 할 수 있는 저장 기법
- Linux에서는 ***"mdadm(Multi Disk ADMin)"*** 명령어를 통해 RAID 구성을 할 수 있다. 패키지 설치가 필요하다.
- RAID 구성 전, ***대상 디스크는 "파티셔닝"이 필수***로 되어 있어야 한다! 그렇지 않으면 추후 마운트 시 문제 발생한다.

1. RAID 0 : "Striping" 기법. 디스크를 합쳐서 용량을 크게 만든다. 속도 ↑, 안정성 ↓, 공간효율 100%
   - 디스크가 하나라도 Fail 발생하면, 모든 데이터가 유실된다.
2. RAID 1 : "Mirroring" 기법. 디스크를 복제해서 이중화한다. 안정성 ↑, 쓰기 속도 약간 ↓, 읽기 속도 약간 ↑, 공간 효율 50%
   - 디스크 하나가 Fail 발생해도 데이터는 안전하다.
3. RAID 5 : "Parity Bit" 기법. 디스크 1개를 검증용으로 만든다. 안정성 ↑↑, 최소 3개의 디스크 필요, RAID 1 보단 효율성 ↑


   ※ Parity Bit ~ 디스크에 저장된 비트의 갯수를 짝수 or 홀수로 맞춰주는 알고리즘
5. RAID 6 : "한 쌍의 Parity Bit" 기법. 디스크 2개를 묶어 검증용으로 만든다. 안정성 ↑↑↑, 최소 4개의 디스크 필요,
   - 디스크 2개가 Fail 발생해도 데이터는 안전하다.
6. RAID 10 : ①RAID 1 로 묶인 디스크를 다시 한번 ②RAID 0 으로 묶는다. 최소 4개의 디스크 필요. RAID 0 보단 안정성 ↑↑
7. RAID 16 : ①RAID 1 로 묶인 디스크를 다시 한번 ②RAID 6 으로 묶는다. 최소 8개의 디스크 필요. 안정성 ↑↑↑↑↑


### mdadm 명령어로 RAID 구성하기
1. 디스크 파티션 구성하기
   <img width="887" height="462" alt="image" src="https://github.com/user-attachments/assets/efdb11ea-0fe5-4361-be17-2ee9d15150c0" />


   ```
   1. fdisk /dev/<파티션 되지 않은 디스크 볼륨 이름. sdb, sdc 등등...>
   2. Command (m for help): 뜨면 → n(new) / d(delete) / l(list-up) / p(print) / w(write)
   ```
   > n 입력하고, 보통은 설정을 별도로 해줄 필요가 없기 때문에 Default 값 유지해서 생성한다. 다 하고 w로 저장.


2. RAID 구성 잡아주기
   ```
   mdadm --create /dev/<만들고자 하는 RAID 디스크 이름> --level=<RAID 타입> --raid-devices=<디스크 갯수> <대상 디스크> <대상 디스크2> ...
   ```

3. RAID 구성 잡은거 확인
   ```
   mdadm --detail 또는 -D /dev/<만들고자 하는 RAID 디스크 이름>
   ```
   > 만약 제대로 RAID를 구성했을 경우에도 부팅 과정에 오류나는 경우가 있다. 그럴때는 mdadm --run 명령을 쳐주자.


4. 잡은 RAID 디스크를 파일 시스템 타입 정의
   ```
   mkfs -t ext4 /dev/<만들고자 하는 RAID 디스크 이름>
   ※ 리눅스에서 추가 스토리지를 구성할 때 대부분 ext4 형태로 디스크를 포맷시킨다!
   ```


5. 포맷 완료된 디스크를 마운트 포인트에 마운트 처리
   ```
   mount /dev/<포맷 완료된 RAID 디스크 이름> <대상 경로>
   ※ 정상적으로 마운트 되면, 구성이 잘 되었는지 최종 확인을 위한 "lost+found" 이름의 빈 폴더가 생성된다. 
   ```


6. 시스템 재시작해도 설정이 없어지지 않도록 부팅 테이블에 등록
   - vi /etc/fstab
     > 잊지 말자! 이거 안해서 잘못됐다 생각하고 삽질했었다.
