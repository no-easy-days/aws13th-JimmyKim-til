## 📝 배운 내용 (Today I Learned)


### WORDPRESS
- 웹 서버(HTTP Apache Server)와 달리 DB와의 통신을 통해 결과물이 변동된다.
- ***FrontEnd & BackEnd 라는 경계 없이*** 동적 웹 페이지를 만들 수 있는 하나의 소프트웨어. (PHP 기반)

1. SELinux 비활성화 및 재시작
   ```
   sed -i s/SELINUX=enforcing/SELINUX=disabled/g /etc/selinux/config
        ↑ ↑   <대상문자열>     <바꿀문자열>     ↑     <대상 파일>
        │ 치환시작(Subtitute)                  해당하는 모든걸 바꿔(Global)
        │
     실제 파일에 반영해라.(in place)
   ```
   > 대상 파일에서 특정 문자열을 찾아서 치환 명령. vim으로 까서 수정하는 짓을 안할 수 있다!
     

2. 서버단 ~ WORDPRESS와 연동할 DB 서버 구축을 위한 패키지 설치
   - yum install -y mariadb-server


     ※ 수업에서는 DB서버도 같은 곳에 다 설치 했지만, 당연히 분리해서 구성할 수 있다.

3. MariaDB 데몬 활성화
   - systemctl enable --now mariadb

4. 데몬 정상 여부 점검
   - ss -tnlp → Socket Statistics, 네트워크 소켓 정보 확인. ***포트 정보 조회 가능***, netstat보다 빠름!
     ※ 필터옵션 ~ t : tcp 통신, n : DNS 말고 숫자 형태로 알려줘, l : Listen인 애들만, p : 프로세스 이름.

5. 정상 확인되면, DB 접속
   - mysql_secure_installation


     ※ 계정 설정 및 테이블 등등의 초기 구성 세팅 명령어. 해당 명령 종료 및 이후 접속에서는 SQL 치듯 하면 됨.
   ```
   grant all privileges on <DB이름>.<대상테이블(*도 가능)> to <유저명>@'<어떤호스트에서 접속하는지(%가 디폴트)>' identified by <비밀번호>;
   ↑        ↑                                                                                   ↑   
   권한  모든 권한을 준다.                       버그인지 모르겠지만, 가끔 localhost에서 안되는 경우 발생함. 그때는 'localhost' 추가해준다.        
   ```
   
   - mysql -h localhost -u root -p<설정할 비밀번호>


     ※ 비밀번호 띄우지 말고 p 뒤에 바로 붙여서 씀.


6. PHP 7 버전 이상을 맞춰주기 위해 레포지토리 변경 & 패키지 설치
   - yum install -y epel-release
   - yum install -y https://rpms.remirepo.net/enterprise/remi-release-7.rpm


     ※ 이건 누군가가 오픈소스 느낌으로 올려둔 "레미레포지토리"에서 rpm 다운 받는 행위
   - yum install -y yum-utils
   - yum-config-manager --enable remi-php73


     ※ yum-utils의 기능이며, 아까 다운받은 "레미레포지토리"의 php73 버전 정보를 등록하는 행위. (버전 맞춰주려고 하는 짓)
   - yum install -y unzip wget
   - yum install -y https://wordpress.org/latest.zip


     ※ 여기까지하면 설치가 완료되는 거고, 그 뒤로는 압축 해제 하고 ***웹 루트 디렉토리(/var/www/html)로 옮기면 됨.***
       > 웹 루트 디렉토리 수정 방법 : vi /etc/htttpd/conf/httpd.conf

7. 설치 완료된 WORDPRESS의 설정값 잡아주기
   - mv wp-config-sample.php wp-config.php → 기존 샘플을 주는데, 이걸 가지고 컨피그 파일 만들어야 함.

8. 컨피그값도 수정 잘 되었으면, 이제 동작 시작.
   - systemctl enable --now httpd
