### virt-builder
- OS 설치용 iso 파일을 가지고 디렉토리에 완전 초기 설정부터 Guest OS를 설치하는 것이 아닌, image 형태로 설치한다.
- 내가 원하는 정보를 넣어줄 수 있다! 이를 위해 virt-builder 명령어 사용이 필요하다.
  
1. 패키지 설치
   ```
   yum install -y libguestfs-tools libguestfs libguestfs-xfs

   ※ virt-builder --help → 내가 활용할 수 있는 파라미터값 다 보여준다. 시작은 여기서부터.
   ``` 

2. 가장 기본이되는 이미지 파일 생성
   ```
   virt-builder centos-7.8 --size 7G --format=qcow2 -o /remote/vm/cent7.img

   ※ centos-7.8 이미지를 베이스로, 크기가 7GB인
      qcow2 형식의 디스크를 만들어서 /remote/vm 경로에 cent7.img 로 저장을 시키겠다.
   ```

3. 생성한 이미지 파일을 바탕으로 VM 설치
   ```
   virt-install --name cen1 --vcpus=2 --ram=2048 --network bridge=br0 --import --disk=/remote/vm/cen1.img --noautoconsole --graphic=vnc

   ※ cen1 이름으로 코어수 2개, 메모리 2GB, 브릿지 대역을 쓰고 cen1.img 을 이미지 그대로 가져다 설치한다.
      virt-viewer는 켜지 않고, VNC 형태로 VM을 조회할 것이다.
      여기서 중요한건, "--import" 설정이다. img 그대로 가져다 박겠다는 명령이다!!!
   ```


※ ***virt-builder는 guest VM의 파일 시스템을 직접 수정***하기 때문에, VM 실행 전에도 mkdir 명령어 사용이 가능하다!


### virt-customize
- virt-builder로 생성한 이미지 파일에 내 입맛대로 설정값을 추가한다.
- virt-builder랑 명령어 완전 동일하다. virt-builder로 생성할 때 놓친게 있다면 이 방식으로 추가하자.

1. 기본 설정 적용하는 방법
   ```
   virt-customize -a cent-template.img --selinux-relabel \
   --upload /etc/yum.repos.d/CentOS-Base.repo:/etc/yum.repos.d/CentOS-Base.repo \
   --firstboot-command 'systemctl disable --now firewalld' \
   --firstboot-command 'sed -i s/SELINUX=enforcing/SELINUX=disabled/g /etc/selinux/config' \

   ```

※ ***--upload 명령은 기본적으로 Overwrite*** 이다!


### SSH에 대한 고찰
- ssh는 기본적으로 Server-Client 형태로 접속 기능을 제공하고, "Public-Key" Authentication 방식이다.
  1) **Server** : 각 계정(root든 user1이든)의 ***홈 디렉토리/.ssh/Authorized_keys 파일에 "Public-Key"값 저장***하고 있다.
     - openSSH를 쓴다면 ***파일명은 "Authorized_keys" 맞춰줘야한다!*** 바꾸고 싶다면 sshd_config 수정 필요하다.
  3) **Client** : 각 계정의 ***홈 디렉토리/.ssh/known-hosts 디렉토리 하위에 "Private-Key"를 저장***하고 있다.


- Server ~ Client 간 Public Key 값이 같다면 접속이 가능하다. 어디서 생성했던, Server는 Private Key는 상관 안쓴다!
- ***Public Key는 "ssh-keygen" 명령을 통해 생성*** 가능하다.
  ```
  ssh-keygen -t rsa -b 4096 -N "" -f ~/.ssh/id_rsa -q

  ※ -t rsa : rsa 방식으로 / -b 4096 : 4096 바이트로 / -f ~/.ssh/id_rsa : 저 경로에 파일 저장 / -q : 결과 출력 X
  ```
- Client에서 ssh 접속 시 Private-Key 값을 지정하려면 "-i ~/.ssh/id_rsa" 파라미터 값을 추가해주면 된다.
