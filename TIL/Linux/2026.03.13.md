### virt-istall
- KVM 위에 Guest VM을 CLI 명령을 통해 생성한다.
  ``` 
  virt-install --name <이름> --vcpus=<CPU코어수> --memory=<메모리용량KB> --os-variant=<OS종류> --cdrom=<iso 파일 저장 경로>
  --network <(network / bridge)=(Interface ID) --graphic=vnc --noautoconsole --disk path=<VM이 저장될 경로>,size=<Max Capacity값>

  ※ 필요 시 virt-install --help 명령어를 통해 파라미터 값 확인할 수 있음. 
  ```


### virsh
1. virsh list
   - 호스트에 설치된 vm 조회. --all 입력을 통해 꺼져있는 vm도 조회한다.

2. cd /etc/libvirt/qemu
   - VM을 정의하는 파일인 ***".xml"***이 존재한다. 
   - VM이 지워지면 매핑되는 xml 파일도 따라 삭제된다.
  
3. virsh define <VM이름>.xml
   - xml 파일을 참조하여 VM을 새롭게 정의하겠다.
  
4. virsh start <VM이름>
   - 정의된 VM을 실행하겠다. (list --all에서 조회하고 입력하는 것이 실수 안할듯)

5. virsh destroy <VM이름>
   - 정의된 VM을 끄겠다.

6. virsh undefine <vm이름>
   - 정의되었던(이미 등록된) VM을 삭제하겠다.

7. virsh console --domain <vm이름>
   - 도메인으로 직접 접속하겠다. (SSH 접속이 아닌 직접 터미널에 들어가는 개념)

8. cd /etc/libvirt/qemu/networks
   - 인터페이스 ID 별로 xml파일이 생성되어 있다.

9. virsh net-list -all
   - 저장된 네트워크 인터페이스 목록을 조회한다.
  
10. virsh net-define <Interface ID>
   - 인터페이스 ID를 등록한다.

11. virsh domiflist <vm이름>
   - 해당 VM의 인터페이스 정보를 조회한다. (유사하게, "domifaddr" 을 입력하면 IP를 조회할 수 있음!)

12. virsh detach-interface <vm이름> network <대상Interface ID> --persistent
   - 설정된 인터페이스를 제거한다. (전체 삭제를 하려면, 인터페이스 ID를 빼면 됨)

13. ***virsh attach-interface <vm이름> (network 또는 bridge) <인터페이스 ID> -model virtio --persistent***
   - 대상 vm에 네트워크 인터페이스를 추가한다.
   - ***"브릿지 또는 네트워크 이더넷" 선택*** 여기서 설정 가능하다!!! 혹시 실수하지 않게 주의하자.
   - 외부망으로 통신하게 만들려면 당연히 브릿지(br0)를 지정해줘야한다!!!


#### 그 외 명령어는 위의 명령어와 유사하다. 필요 시 virsh --help로 확인한 후 파라미터 값 추가하자.


- CLI를 통한 VM 마이그레이션 명령
  ```
  virsh migrate --unsafe --live --persistent --undefinesource --verbose <VM대상이름> qemu+ssh://<목적지주소>/system

  ① undefinesource : 기존 VM은 없애고, 옮겨갈 VM만 존재하도록 명령 (Cut & Paste)
  ② verbose : 진행상황을 콘솔에서 조회

  ※ 목적지 IP만 가지고는 불가능하며, 목적지 서버에서 원래 VM 저장된 서버의 HostName을 알고 있어야한다!!! /etc/hosts 수정 필요!!!
  
  ```
