### Linux 개요 및 기초
1. 리눅스
   - Debian 계열 : 범용성이 높다. (패키지 : deb, 패키지 관리자 : apt)
     → 종류 : debian, **ubuntu**, kali linux 등
   - RedHat 계열 : 기업 친화적이다. (패키지 : rpm, 패키지 관리자 : yum, dnf)
     → 종류 : Centos(무료버전), RHEL, Fedora 등
   - 그 밖에 : SUSE (패키지 : rpm, 패키지 관리자 : zypper)


2. Repository (CentOS 7 기준 /etc/yum.repos.d)
   - 일종의 "책꽃이"
   - 설치하고 싶은 패키지들("책")가 있는 저장 공간.
   - mirrorlist : 미러 사이트들 목록(패키지들이 복제되어 있는 저장소. 영문 주소로 구성되어 있다.)
     ※ ***DNS 서버 명시가 반드시!! 필요***하다. cat /etc/resolv.conf로 확인.
   - base-url : 단일 미러 사이트
   - gpgcheck : 무결성 확인. gpgcheck=1이 정상이며, 원본과 대조하여 다르면 차단 or 경고를 뿌림.


3. 디렉토리 구조(/)

     ※ pwd : Present Working Directory. 현재 작업중인 디렉토리를 확인


     ※ ***. : 현재 디렉토리, .. : 상위 디렉토리***    


4. 리다이렉션 (>, >>, <, <<)
   - 입력이나 출력의 ***방향***을 바꿔준다.
   - < 이나 <<은 입력의 방향을 / > 이나 >>은 출력의 방향을 바꿔준다.
   - ***>> : 기존 내용에 이어서 내용 추가, > : 덮어 쓰기***


5. vi 편집기

6. 기본 리눅스 세팅 시 확인해야할 것
   - vi /etc/sysconfig/network-scripts/<이더넷>
     - BOOTPROTO="none" : DHCP 비활성화 (YES : 활성화)
     - ONBOOT="yes" : 켜져 있는지 확인!!
     - DEFROUTE="yes : 디폴트 라우팅 경로 적용 확인!!!
   - vi /etc/selinux/config
     - SELINUX=disabled로 변경하기. 재부팅 필요하며, 보안 설정이라 짜증나서 바꾸는게 나음
   - systemctl stop firewalld & systemctl disable firewalld
     - 방화벽 중지 & 설정 해제
