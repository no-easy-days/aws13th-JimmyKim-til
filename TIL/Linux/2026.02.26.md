### Proxy Server
- 다수의 ***클라이언트들을 대신**한다. "주체"가 누구인지가 포인트이다!
- 캐싱 / 클라이언트가 외부로 나가는 것도 관리한다.
- Proxy Server가 없다면, 각 클라이언트가 서버로 개별적으로 요청하는 상황이 된다. Proxy Server가 있으면, 한번만 하면 캐싱된다.

2. TOMCAT 서버 구축
- Web Application을 배포하는 WAS 서버 SW. 개발환경은 **JAVA**
  - wget, unzip, java-11-openjdk 패키지 설치
    ※ 정확한 패키지 명을 모르겠으면 yum search 를 쓰면 된다.
  - tomcat 설치
    ※ 웹사이트 들어가서 설치한다. (yum을 쓰지 않고, 링크 복사 후 wget + unzip + chmod 777 처리함. 방식은 내 맘대로)
       - 주소 : http://tomcat.apache.org/download-10.cgi
   
         
    ※ /webapps : 웹 애플리케이션의 배포를 위한 루트 디렉토리
 
    
    ※ /bin/catalina.sh : Tomcat의 메인 실행 파일. foreground에서 동작한다. ← 추후, Docker, K8s에서는 foreground에서!
 
    
    ※ /bin/startup.sh : Background 실행 쉘 스크립트 (ss -tnlp로 정상 여부 확인)
 
    
    ※ /bin/shutdown.sh : Background에서 실행되고 있는 tomcat 중지 쉘 스크립트
 
    
    ※ jsp 파일 : /webapps/ROOT/ 하위에 저장하면 된다. 별도 데몬 재시작 필요 없음.
 
    
    ※ war, jar 파일 : /webapps/ 하위에 저장하면 자동으로 폴더 형태로 압축 해제된다.

  - DB 연동을 위한 ***JDBC 설치***
 
  - 
    ※ JAVA로 DB를 연결하기 위한 라이브러리, jar 파일 형태로 설치 가능하며 웹에서 다운로드하면 된다.
       - /lib 경로 하위에 jar 파일 다운로드 후 tomcat 중지 & 시작 명령 필요
