### LVM
- ***L***ogical ***V***olume ***M***anager
- 물리적인 디스크/파티션을 내가 원하는 방식과 용량의 가상 디스크로 구성할 수 있다.
- 다수의 물리 디스크를 가지고 ***"PV(Physical Volume)"*** 를 구성하고, ***PV를 바탕으로 "VG(Volume Group)"*** 를 구성한다.
- 물리 디스크는 물론이고, 파티션들도 PV가 될 수 있다.
  <img width="827" height="502" alt="image" src="https://github.com/user-attachments/assets/316d7f5c-0f50-462c-a917-428ddccc3b4d" />


  1. LVM 패키지 설치
     ```
     yum install -y lvm2
     ```
  2. PV 지정 및 선언
     ```
     pvcreate /dev/<디스크/파티션이름>   ~   대상이 되는 디스크 or 파티션마다 지정해줘야 한다.
     pvscan : 생성된 PV 정보 간략 조회
     pvdisplay : 생성된 PV 정보 상세 조회
     ```
  3. VG 지정 및 선언
     ```
     vgcreate <VG이름> <대상 PV 경로>
     vgscan : 생성된 PV 정보 간략 조회
     vgdisplay : 생성된 PV 정보 상세 조회
     ```
  4. LV 지정 및 선언
     ```
     lvcreate --name <LV이름1> --size <용량> <대상 VG 이름> : 원하는 용량 크기로 LV 생성
     lvcreate --name <LV이름2> --extents +100%FREE <대상 VG 이름> : 위에서 생성되고 남은 용량 활용해서 다른거 만들기
     lvscan : 생성된 LV 정보 간략 조회
     lvdisplay : 생성된 LV 정보 상세 조회

        ※ ls -al /dev/<VG이름>/*   ~  만들어진 LV를 심볼릭 링크 형태로 확인할 수 있다.
     ```
  5. 파일시스템 선언 및 생성 & 마운트
     ```
     mkfs -t ext4 <LV경로>   ~   경로는 보통 /dev/<VG이름>/<LV이름> 형태
     mount <LV경로> <마운트 포인트 경로>
     ```


  ※ 용량 확장 시)
  1. VG 용량 증설
     ```
     vgextend <VG이름> <증설된 디스크 경로>   ~   추가할 디스크도 당연히 PV로 선언되어 있어야한다!
     ```
  2. LV 용량 증설
     ```
     lvextend --size <지정할 용량> <LV경로> : 지정하고자 하는 용량으로 LV 용량 확장
     ```
  3. 파일시스템 재구성(리사이징) ~ ext4 기준
     ```
     resize2fs <LV경로>  :  Default는 추가할 수 있는 LV 용량 만큼 리사이징한다.

     ※ 포맷을 하는건 아니다. 리사이징을 해줘야 실제 리눅스 OS에서 인지를 할 수 있다.
     ```


  ※ 용량 축소 시)
  1. ***mount 해제*** ← 주의!!!
     ```
     umount <마운트 포인트 경로>
     ```
  2. 파일시스템 체킹 ~ 섹터에 산재된 데이터를 정리해주는 느낌으로 이해하자.
     ```
     e2fsck -f <LV경로>
     ```
  3. 줄일 용량만큼 파일시스템 재구성
     ```
     resize2fs <LV경로> <목표 용량>
     ```
  4. LV 용량 축소
     ```
     lvreduce --size <목표 용량> <LV경로>
     ```
  > 용량 확장이랑 거꾸로 간다. lv 확장 → 리사이징 vs 리사이징 → lv 축소.
