## Python 실습 과제 막혔던 부분들 정리

1. 파일 입출력 시 반복문을 활용하는 것이란?
  - read() 또는 readlines() 를 사용하면, 대상이 되는 파일 "전체"를 메모리에 올린다.
  - for 문을 활용하여 문자열을 읽어오면, 파일의 문자열을 "한줄 씩 읽어와서 처리"하기 때문에 메모리 절약에 좋다.
    > 파일 객체 "f" 라는 녀석 자체가 "파일 객체" 이자 "이터러블"이며, 다음 줄을 기억하고 반환하는 상태 기반 이터레이터.
  ```
  lines = f.readlines()
  result = []
  for i in lines:
	result.append(i.strip())  # 이렇게 하면, 파일을 다 읽어서 한줄한줄 처리
  #반면에, for 문으로 처리하면
  line.strip() for i in f:  # 한 줄에 끝내기도 하고, 메모리도 절약할 수 있다!
  ```

3. 파일 복사 & 붙여넣기 시 주의할 점
  1) 파일 읽어올 때 청크 사이즈를 지정하는 이유
     - f.read(사이즈) : 이렇게 인자값을 넣어주면, 지정한 사이즈씩 읽게 된다. 
     ```
     # 예시 코드
     with open(파일, 'rb') as f:
	   while True:
		 chunk = f.read(4096)
		 if not chunk:
			 break
     # 이런 형태로 많이 사용한다. 읽을 데이터가 있는지 없는지를 확인해서 읽기 종료 시점을 체킹함
     ```
     
     - 한번에 파일을 전부 읽지 않고, "스트림" 방식으로 청크 사이즈 만큼만 읽어서 바로 써주기 위해.
     - 메모리 절약을 위해. 파일이 얼마나 큰지를 모르니까. 대용량 파일 복사도 가능하도록.
       > 스트리밍 I/O를 위해 필요한 개념인 것 같다. 아직은 잘 안와닿지만 이해해두자.
  2) 청크 사이즈 소소하게 활용하기
     ```
     copied_size += len(chunk)
     ```
     - 이런 방식을 써서 얼마나 복사되었는지를 누적하고, 이걸 가지고 진행율 표시할 때 많이 쓴다.

  4. while 문 활용하여 특정 조건 올때 까지 무한 루프 돌리기
     ```
     while True:
	 if 조건:
		프로그램 수행
		break
     ```
     - 무한 루프 돌리다가 if 문으로 break 걸어줘서 멈추게 하는 로직이 제일 깔끔함.
