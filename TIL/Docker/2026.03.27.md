## Dockerfile ★★★★★
- 일종의 명세표, 내가 원하는 옵션(Desired Option)이 담겨있다.
<img width="896" height="427" alt="image" src="https://github.com/user-attachments/assets/59ef59d4-e0fd-4941-8679-cd9acb89796c" />


- Dockerfile 및 의존성 패키지들을 적절히 생성한 뒤, 아래 명령어로 image를 생성한다.
  ```
  docker build -t <레지스트리>/<레포지토리>:<태그> -f <Dockerfile 경로>

  ※ -t : 태그를 달아서 이미지를 빌드하겠다.
     레지스트리 : "도서관" / 레포지토리 : "책꽂이" / 태그 : "책"
     레지스트리는 생략 가능하다. 대신 명시하지 않으면 기본값은 docker.io(Docker 허브)


  docker login -u jaehyunjimmykim(내 username)
  docker push <레지스트리>/<레포지토리>:<태그>
  
  ※ Docker 허브에 내가 커스터마이징한 이미지를 PUSH 하고 싶다면, 레지스트리까지 잘 명시해야한다.
     레지스트리는 username이 된다.
  ```


### Dockerfile은 단순 Script가 아니라, 한줄 한줄 Layer를 쌓아가며 만든다는 느낌으로 이해하자.


## Dockerfile 주요 명령어


### 1. FROM : 기본 기틀(Base)로 쓸 이미지 지정
- 운영체제가 될수도 있고 전문가들이 만들어둔 이미지가 될 수도 있다.


### 2. WORKDIR : 작업 디렉토리 지정
- ***"docker exec -it" 명령어 치면 제일 먼저 접속***하게 되는 디렉토리.
- cd 명령처럼 사용하면되고, 재지정 전까지는 영구적으로 유지된다.
- 만약에 /dir1 에서 작업을 하고 /dir2에도 작업을 해야한다? → ***여러 번 입력해서 이동 시키면 된다!***
  ``` 
  WORKDIR /dir1
  작업(RUN) ~~~~
  WORKDIR /dir2
  작업(RUN) ~~~~
  ```
  > Dockerfile은 한줄 한줄 Layer를 쌓는다는 것을 잊지말자. 순서가 생각보다 중요하다.


### 3. COPY : Host의 파일이나 디렉토리를 Container 이미지에 복사
- 파일이라면 파일로, 디렉토리라면 디렉토리로 복사하는 것이 이슈가 발생하지 않는다.
  > /test/* 로 쓰면 파일들이 될 것이고, /test 로 쓰면 디렉토리가 될 것이다!!


- 항상 WORKDIR을 COPY 이전에 선언하자. (경로 꼬이지 않게)
- Host에서 ***Dockerfile이 생성되어있는 경로가 가장 최상위 디렉토리***이다. 그 밖에 있는건 불가능!


### 4. RUN : 이미지 Build 단계에서 수행되는 Command
- 하나의 Layer에서 여러개의 명령을 수행하고 싶다면 && 로 묶어서 길게 쓸 수 있다.
- cd 같이 일시적인 명령어는 Layer가 바뀌면 초기화된다. (의미가 없어져버린다)
- 대신에 패키지 설치 또는 Foreground 외에 Background로 돌려야 하는게 있다면 RUN으로 하는 것이 좋겠지.


### 5. CMD : 컨테이너 Run 단계에서 수행되는 Command
- ***"docker run "*** 명령어로 컨테이너를 실행시킬 때 우리가 원하는 ***프로세스가 foreground로 동작***하게 만든다.
- 여러번 수행은 가능하지만, 맨 마지막꺼만 적용되기 때문에 의미가 없다. 한번만 쓰자.
  ```
  CMD ["   ", "   "]
  # 띄어쓰기마다 쉼표로 구분하고 쌍따옴표로 묶어주면 된다.
  ```
  > 팁! 처음 image 만들 떄 CMD ["sleep", "infinity"] 를 입력하면 컨테이너가 Stop 되지 않는다.


- ***"docker ps -a"*** 에서 ***"COMMAND"*** 를 보면 내가 입력한 Command를 확인할 수 있다.


### 6. ENTRYPOINT : 컨테이너 Run 단계에서 Argument(인자값)를 받아오게 만드는 Command
- CMD와 유사하지만, ***"docker run"*** 명령어에서 ***<레지스트리>/<레포지토리>:<태그> 뒤에 인자값이 필요할 경우*** 사용.
- CMD랑 비교해서 입맛에 맞게 사용하면 된다. 상황에 맞춰서 쓰면 됨.


### 7. ENV : 환경 변수 선언
- mysql 같이 필수 환경 변수가 필요한 경우가 있을 때 사용한다.
- ***"docker run -e"*** 명령어로 했던 걸 사전에 해주는 느낌이다. 이것도 상황에 맞춰서 쓰면 된다.


### 8. 디버깅 팁 (잊지 말자)
- docker image inspect <레지스트리>/<레포지토리 이름>:<태그>
  - 내가 Dockerfile에서 설정했던 값들을 찾을 수 있다. | grep 써서 적절히 찾거나 스크롤로 확인하자.
- docker inspect <컨테이너 이름>
  - ENTRYPOINT 같이 인자값을 부여해야 한다면, 제대로 받아왔는지 여기서 확인할 수 있다.
