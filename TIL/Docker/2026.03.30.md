## 1. --link
- 대표적인 예시로 Application 과 DB 연동 시, DB가 IP를 부여받아 외부 노출 받는건 보안 상 별로 & 번거로움.
- ***Container의 "이름을 IP 주소처럼" 활용***한다.
  > --network 도 있긴한데, 일단 --link로 개념을 이해해보기


```
docker run --name <컨테이너 이름> ~~~(run 명령어) --link <링크할 컨테이너 이름> <레지스트리>/<레포지토리>:<태그>

※ 원리 : /etc/hosts 밑에 링크할 컨테이너 Host 정보를 등록해준 것이다. (미니 DNS 같이 이름으로 찾아감)
```


## 2. 개인 레지스트리 (private registry)
- 나만의 Docker Hub를 내 로컬 환경에 구축한다. 레지스트리 주소를 <대상IP주소>:<포트> 형식으로 등록.
- 레지스트리 주소에 맞게 컨테이너 이미지를 빌드해야 Push가 가능하다.

  
```
docker run --name <이름> --restart=always -dp <접속시킬포트>:5000 -v /registry:/var/lib/registry/docker/registry/v2 registry:latest

※ 컨테이너 중지됐을 때 자동 재시작 설정을 걸어줬다.
  - /var/lib/registry/docker/registry/v2 : 컨테이너 이미지가 push 되었을 때 저장 경로
```


- 기본적으로 docker push는 https 통신을 하게 되어있다. 아래 두가지 방법 중 하나로 구현해야 한다.
  - TLS 인증서를 붙이기 (당연히 인증서 필요)
  - http 통신으로 변경 설정 적용
    ```
    vi /etc/docker/daemon.json
    
    {
        "insecure-registries" : ["<대상IP주소>:<포트>"]
    }

    ※ 클라이언트(Push)로서의 설정. Docker는 시작 시 해당 파일을 읽어들인다!
    ```


- habor 나 nexus3 같은 것들이 이러한 기술을 활용한 것이다.
