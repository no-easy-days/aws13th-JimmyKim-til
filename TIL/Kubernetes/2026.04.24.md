## 1. Helm
- node.js 의 npm이 패키지 관리자라고 하면, helm은 K8s의 Manifest Chart 관리자이다.
- `Chart` : Manifest들을 패키징한 포맷.
  ```
  wordpress/
      Chart.yaml               # A YAML file containing information about the chart
      LICENSE                  # OPTIONAL: A plain text file containing the license for the chart
      README.md                # OPTIONAL: A human-readable README file
      requirements.yaml        # OPTIONAL: A YAML file listing dependencies for the chart
      values.yaml              # The default configuration values for this chart
      charts/                  # A directory containing any charts upon which this chart depends.
      templates/               # A directory of templates that, when combined with values,
                               # will generate valid Kubernetes manifest files.
      templates/NOTES.txt      # OPTIONAL: A plain text file containing short usage notes

  ※ 이런식으로 chart가 구성된다. 하나의 App 배포 시 필요한 다양한 yaml 파일이 집약되어있다.
  ``` 
- https://artifacthub.io : 헬름을 다룰 때 제일 중요한 곳, 헬름 차트들이 모여있는 일종의 Hub

- 설치 방법
  - `curl -fsSL -o get_helm.sh https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3`
  - `chmod +x get_helm.sh && ./get_helm.sh`

- 레포지토리 추가하기
  - `helm repo add <지정할 별칭> <레포지토리URL>`, 레포지토리 URL은 https://artifacthub.io에서 찾아온다.
  - github에서 origin 등록하는 것과 비슷한 느낌이다.

- 내 로컬에 받은 레포지토리 조회 및 업데이트(Ubuntu `apt`처럼 주기적으로 실행하자)
  - `helm repo list`, `helm repo update`

- 레포지토리에서 내가 필요한 차트 찾기
  - `helm search repo <내가 찾고 싶은 차트 이름>`, 결과는 `<별칭>/<차트명>` 형식으로 조회된다.

- Release (이미지로 Container들을 띄우듯이, 차트를 가지고 Release를 찍어낸다)
  - `helm install <지정할 Release명> <별칭>/<차트명>`, Release에 맞춰 리소스들의 이름을 적절히 변경해준다.
  - `helm list` : Release 목록 조회, `helm delete 또는 uninstall <Release명>` : Release 삭제.

- #### 차트가 가진 변수 조회방법
  - `helm inspect values <별칭>/<차트명>`, 리다이렉션(`>`) 활용해서 `yaml` 파일로 저장해서 활용하면 된다.
  - 변수를 수정 후 적용해서 Release 생성 시에는 `helm install <Release명> <별칭>/<차트명> -f <yaml파일명>` 
  - https://artifacthub.io 의 Chart에서 `Default values` 메뉴를 활용해서 조회 가능하다.
