## 1. Job
- 일정 시간 동안 실행되는 **"일회성(One-time)"** 작업이나 반복 없이 작업을 완료해야 할 때 활용한다.
- “지정된 작업이 **정상적으로 완료**될 때까지 **Pod**를 실행하도록 **보장**”하는 컨트롤러
- 동시에 여러개의 Pod를 실행할 수도 있고, 성공한 Pod가 일정 갯수에 도달하면 Job을 완료 처리한다.
-  **Job Controller**가 관리하는 Pod들은 ***종료 시점(Exit Code 0)까지 실행***된다.
- 작업 실패 시, Job Controller가 Pod 재시작(일정 횟수까지) 또는 새로운 Pod를 생성하여 작업이 **반드시 완료**되게 한다.
- `yaml` 파일 예시
  ```
  apiVersion: batch/v1
  kind: Job
  metadata:
    name: example-job
  spec:
    completions: 3           # 실행 성공해야할 수치
    parallelism: 2           # 동시에 실행되어야 할 Pod 수
    backoffLimit: 4          # 최대 재시도 횟수(실패 시 새 Pod 재생성)
  template:
    metadata:
      name: example-job-pod
    spec:
      restartPolicy: Never   # Job에서는 보통 Never 또는 OnFailure를 사용
      containers:
      - name: worker
        image: busybox
        command: ["sh", "-c", "echo Hello; sleep 5; echo World"]
  ```
- `kubectl get jobs` : `NAME`, `COMPLETIONS`, `DURATION` 등 정보 조회 가능
- Job이 완료된 후 컨테이너 안에서 생성된 파일이나 결과물이 있다면 PV 또는 외부 스토리지에 저장해야 한다.
- 그렇지 않으면 Pod이 종료되며 바로 삭제되기 때문에 데이터를 잃게됨
- 주요 활용 사례
  - 데이터 마이그레이션, 로그 처리, 이미지 변환, DB 백업, **배치 작업** 등 “**한 번(또는 일정 횟수) 수행 후 종료**”를 원하는 케이스


## 2. Cron-Job
- 지정된 스케줄(시간/주기)에 맞춰 Job(일회성/배치 작업)을 반복 실행할 수 있도록 해준다.
- Linux crontab 스타일의 5개(또는 6개) 필드를 사용하며, 당연히 시간대 설정을 잘 해줘야한다!
- Job과 동일하게 완료(Exit Code 0)가 정상적으로 될 때까지 Pod 실행을 보장한다.
- 한 스케줄의 Job이 아직 완료되지 않았는데 다음 스케줄 시간이 도달하면 ***Job이 중복 실행***될 수 있다.
  - 백오프 설정(`backoffLimit`, `activeDeadlineSeconds` 등)을 통해 중복 처리를 회피한다.
- `kubectl get cronjob` : Cronjob이 마지막으로 실행된 이력 조회한다.
- `yaml` 파일 예시
  ```
  apiVersion: batch/v1
  kind: CronJob
  metadata:
    name: example-cronjob
  spec:
    schedule: "0 2 * * *"                 # 매일 새벽 2시에 실행
    successfulJobsHistoryLimit: 3         # 성공 Job의 히스토리를 몇개까지 보관할 지
    failedJobsHistoryLimit: 1             # 실패 Job의 히스토리를 몇개까지 보관할 지
    jobTemplate:                          # CronJob이 생성할 Job의 템플릿
      spec:
        template:
          spec:
            containers:
              - name: batch-worker
                image: busybox
                command: ["sh", "-c", "echo Hello from CronJob; sleep 5; echo Done"]
            restartPolicy: Never
  ```
- 너무 많은 Job 히스토리를 남겨두면 etcd에 부담이 될 수 있다. 3~10건 정도가 적당하다.


## 3. Helm
- node.js 의 npm이 패키지 관리자라고 하면, helm은 K8s의 Manifest Chart 관리자이다.
- `Chart` : Manifest들을 패키징한 포맷.
  ```
  wordpress/
      Chart.yaml               # A YAML file containing information about the chart
      LICENSE                  # OPTIONAL: A plain text file containing the license for the chart
      README.md                # OPTIONAL: A human-readable README file
      requirements.yaml        # OPTIONAL: A YAML file listing dependencies for the chart
      values.yaml              # The default configuration values for this chart
      charts/                  # A directory containing any charts upon which this chart depends.
      templates/               # A directory of templates that, when combined with values,
                               # will generate valid Kubernetes manifest files.
      templates/NOTES.txt      # OPTIONAL: A plain text file containing short usage notes
  ``` 

- https://artifacthub.io : 헬름을 다룰 때 제일 중요한 곳, 헬름 차트들이 모여있는 일종의 Hub

- 설치 방법
  - `curl -fsSL -o get_helm.sh https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3`
  - `chmod +x get_helm.sh && ./get_helm.sh`

- 레포지토리 추가하기
  - `helm repo add <지정할 별칭> <레포지토리URL>`, 레포지토리 URL은 https://artifacthub.io에서 찾아온다.
  - github에서 origin 등록하는 것과 비슷한 느낌이다.

- 내 로컬에 받은 레포지토리 조회 및 업데이트(Ubuntu `apt`처럼 주기적으로 실행하자)
  - `helm repo list`, `helm repo update`

- 레포지토리에서 내가 필요한 차트 찾기
  - `helm search repo <내가 찾고 싶은 차트 이름>`, 결과는 `<별칭>/<차트명>` 형식으로 조회된다.

- Release (이미지로 Container들을 띄우듯이, 차트를 가지고 Release를 찍어낸다)
  - `helm install <지정할 Release명> <별칭>/<차트명>`, Release에 맞춰 리소스들의 이름을 적절히 변경해준다.
  - `helm list` : Release 목록 조회, `helm delete 또는 uninstall <Release명>` : Release 삭제.

- #### 차트가 가진 변수 조회방법
  - `helm inspect values <별칭>/<차트명>`, 리다이렉션(`>`) 활용해서 `yaml` 파일로 저장해서 활용하면 된다.
  - 변수를 수정 후 적용해서 Release 생성 시에는 `helm install <Release명> <별칭>/<차트명> -f <yaml파일명>` 
  - https://artifacthub.io 의 Chart에서 `Default values` 메뉴를 활용해서 조회 가능하다.
