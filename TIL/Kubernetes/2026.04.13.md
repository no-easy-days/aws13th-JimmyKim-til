## 1. Kubernetes 컴포넌트 개요
<img width="1117" height="560" alt="image" src="https://github.com/user-attachments/assets/d2b35741-e559-4f4d-9a08-1aed5f5d9352" />


- Master 노드에만 존재하는 컴포넌트
  
  (1) ***kube-apiserver*** (≒ 지배인)
    - 내/외부의 모든 요청을 주고받는 서버. contoller나 scheduler, proxy의 주시 대상. 

  (2) ***kube-scheduler*** (≒ 직원)
    - 생성될 리소스들을 ***어떤 노드에 '배치'할지*** 결정(스케줄링)

  (3) ***kube-controller*** (≒ 하우스키퍼)
    - 다양한 리소스에 대한 여러가지 컨트롤러들이 존재한다.
    - 원하는 상태(Desired state)에 현재상태(current state)가 수렴하도록 ***지속적으로 모니터링. 문제가 생기면 고치거나 리소스를 재생성.***

  (4) ***etcd*** (≒ 장부)
    - 클러스터 및 모든 리소스에 대한 정보를 key & value 쌍 형태로 저장하는 일종의 DB.


- 모든 노드(Worker & Master)에 존재하는 컴포넌트

  (1) ***kubelet***
    - 노드 관리자. 실질적으로 각 워커 노드에 존재하는 리소스 관리

  (2) ***kube-proxy***
    - 노드 안과 밖을 넘나드는 수직 트래픽 관리.



## 2. VM 환경에서 K8s 구축해보기
 
### (1) Container Runtime(***containerd***) 세팅
  - ``` apt install -y curl gnupg2 software-properties-common apt-transport-https ca-certificates``` : 필요 패키지 설치
  - ```curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmour -o /etc/apt/trusted.gpg.d/docker.gpg``` : gpg key 다운로드
  - ```add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"``` : K8s 레포지토리 추가
  - ```apt update -y && apt install -y containerd.io``` : containerd 패키지 설치
  - ```containerd config default | sudo tee /etc/containerd/config.toml``` : containerd의 구성 기본값을 config.toml 파일로 지정.
  - ```sed -i 's/SystemdCgroup=false/SystemdCgroup=true/g' /etc/containerd/config.toml``` : K8s 노드의 시스템 안정성을 높이기 위한 필수 설정
  - ```systemctl enable --now containerd``` : containerd 서비스 활성화


### (2) Linux Kernel 설정
  - ```swapoff -a``` : 스왑메모리 중지 & 비활성화
  - ```/etc/modules-load.d/containerd.conf``` : 컨테이너 구동 시 오버레이 네트워크 구성을 위한 Kernel Level에서의 기반 기능 활성화 
    ```
    vi
     
    overlay
    br_netfilter
    ```
  - overlay
    - 컨테이너 이미지는 여러 레이어로 구성되어 있으며, 이 레이어들을 하나로 합쳐서 실제 파일 시스템처럼 보이게 해준다.
    - 이게 없으면 컨테이너 이미지를 불러오거나 실행할 수 없다.
  - br_netfilter
    - K8s의 Pod끼리 통신할 때 Bridge Network를 통과하게 되는데, 이때 iptables 규칙이 적용되도록 만들어준다.
    -  즉, 네트워크 필터링과 통신을 정상적으로 처리하기 위한 필수 세팅.

      
  - ```/etc/sysctl.d/kubernetes.conf``` : k8s가 네트워크 트래픽을 정상적으로 라우팅 & 필터링하도록 Kernel Level에서의 설정
    ```
    vi
     
    net.bridge.bridge-nf-call-ip6tables = 1
    net.bridge.bridge-nf-call-iptables = 1
    net.ipv4.ip_forward = 1

    ※ br_netfilter 모듈의 "기능"을 켰다면,  그 기능을 "어떻게 동작시킬지" 세부 스위치를 올리는 것.
    ```

 
  - ```modprobe overlay```, ```modprobe br_netfilter```  : 앞서 세팅한 모듈 로드
  - ```sysctl --system``` : Linux Kernel Level 단 설정 적용


### (3) K8s 설치
  - ```
    echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] \
    https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /" | sudo tee /etc/apt/sources.list.d/kubernetes.list
    ```
  - ```
    curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.30/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
    ``` 
    > K8s 설치를 위한 레포지토리 업데이트 작업


  -  ```apt update -y && apt install -y kubelet kubeadm kubectl``` : k8s 및 필요 패키지 설치
     - ***kubectl*** : api요청을 하는 명령
     - ***kubeadm*** : 클러스터를 구성하는 명령어. 
     - ***kubelet*** : 각 노드를 관리하는 관리자.

  
  - ```apt-mark hold kubelet kubeadm kubectl``` : 업데이트 되지 않도록 버전 고정. ***K8s는 버전이 중요***하다.
