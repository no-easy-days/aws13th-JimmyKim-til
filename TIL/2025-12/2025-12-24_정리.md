Git - "분산" (동시성제어 & 충돌제어)
Git : 작업공간 / 준비영역 / 저장소 3가지로 구성
  1) git init : 저장소 초기화
  2) git add : 로컬 -> 준비영역
  3) git commit : 준비영역 -> 저장소

브랜치 : 커밋을 가리키는 포인터(책갈피 느낌)
  1) git branch : 브랜치 추가 (=git switch -c)
  2) git switch : 브랜치 이동

<브랜치 전략(이름 규칙), git flow 또는 github flow가 있음>
main 혹은 master : 배포 가능한 안정 버전
   !!!! 그냥 짜증나니까 git init 하자마자 git branch -M main 때려서 바꿔주자.
develop : 개발 통합 브랜치
feature/ : 새 기능 개발. (ex : feature/account)
bugfix/ : 버그 수정. (ex : bugfix/login-error)
hotfix/ : 긴급 수정. (ex : hotfix/vulnerability)
  
* HEAD : 지금 내가 어디있는지. HEAD는 무조건 한개. "현재 브랜치를 추적"

브랜치 머징 ~ 각자 작업한걸 모아 모아 완성본 만들기.
 - Fast-Forward : main을 feature의 최종적으로 커밋한 버전으로 옮겨버린다.
 - 3-way-Merge : 작업자가 두명 이상일때. 합치고 합침.
  * 결국, merge로만 하면 된다.

원격 저장소 : 클라우드 형태로 저장하는 것
 1) git push : 로컬 > 원격
 2) git pull : 원격 > 로컬
 3) git remote : 원격 저장소 연결.
     - 그전에, git remote add 를 때려서 원격 저장소 추가해야함.
     - git remote -v : 원격 저장소 확인

gh : github에서 만든, 터미널에서 사용할 수 있는 CLI 커맨드
