### 도메인 & 리소스 정의하기
  1) 먼저, 4가지 리소스를 "명사"로 보고, REST 관점에서 정의 시작
  2) 리소스 간 관계 그리기
  ```
  User
 ├─ Post
 │    ├─ Comment
 │    └─ Like
  ```

  - 이런 식으로 일단 관계도를 그려 두기
  3) "기능"이 아니라 "행위"로 접근 - "CRUD"
  4) 댓글 관리에 comments id, 좋아요 관리 likes id를 넣은 이유?
    - 수정 삭제 할때 대상만 건드리기 위해서.

<결론>
me나 쿼리 스트링은 "목록을 조회할 때(GET)" 필터링 용도로만 쓰고, **"무언가를 바꾸거나 지울 때(CUD)"**는 리소스 ID만 URL에 넣고 나머지는 백엔드 로직에 맡기면 된다!
