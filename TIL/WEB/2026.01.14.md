## FastAPI 교재 복습
1. Python Type Hints
  1) name: str ~ name은 문자열. 경로 및 쿼리 파라미터 검증
  2) -> dict ~ 반환값은 딕셔너리. "응답 스키마 문서화"
  3) age: int = 0 ~ age는 기본값이 0 인 정수. "선택적 쿼리 파라미터"
  4) items: list[str] ~ items는 문자열로 구성된 리스트 자료형. "요청 본문 배열 검증"

2. 정규 표현식
  1) re : 정규 표현식을 사용하기 위한 모듈
  2) \d : 숫자 1개, {n} : 반복횟수지정 (예 : \d{4}), . : 아무 문자 1개
     [abc] : 대괄호 내에 있는 문자와 매칭, [a-z] : 범위 지정

3. 동시성과 async / await
  1) 비동기 : 기다리는 시간에 다른 일을 처리하는 방식
  2) I/O Bound 작업 : 프로그램이 직접 계산하는 것이 아닌, 외부에서 데이터가 와야함.
     - 동시성(Concurrency)이 좋다
  3) CPU Bound 작업 : 외부를 기다리는게 아닌, CPU가 직접 열심히 계산해야 함.
     - 병렬성(Parallelism)이 좋다
  4) async : 비동기 함수를 만드는 방법, 기다리는 시간이 있을 수 있다고 알려준다.
  5) await : async def 안에서만 쓸 수 있고, 기다리는 동안 다른 일을 할 수 있게 해줌.

4. 가상환경(venv)
  1) python -m venv .venv 명령어를 통해 가상환경 생성.
  2) source .venv/bin/activate 이후, which python 명령어를 통해 가상환경 적용 여부 판단.

5. 환경변수
  - 코드 외부에 존재하여, 민감한 정보를 코드에 직접 쓰지 않아도 된다.
  - gitignore에 .env 파일 추가를 통해 커밋되지 않을 수 있다.
  - 개발/테스트/운영 환경마다 다른 값을 사용할 수 있다.
  - 운영체제 수준에서 관리되어 여러 프로그램이 같은 값을 읽을 수 있다.

6. FastAPI
  - from fastapi import FastAPI : 클래스 가져오기
  - app = FastAPI() : 앱 인스턴스 생성
  - @app.get() / @app.post() / @app.put() / @app.delete() 를 활용, 데코레이터를 사용한다.
  - async def root(): 경로 작업 함수 정의
  - return {...} : 응답 반환
  - git bash 터미널에서 fastapi dev ~~~.py : 개발용 fastAPI 서버 실행
