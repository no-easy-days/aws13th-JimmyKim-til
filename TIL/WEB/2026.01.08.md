## REST API 작성 & 설계 중 유의 사항
1. GET 메서드 만들 때
  1) Request Header 內 Content-Type
     - Request Body가 있을 경우 명시하면 된다. 그 외에는 필요없음.
  2) Path Parameter
     - 뭔가 특정 경로(예를 들어, 개별 페이지)가 필요할 때 사용하면 된다.
  3) Query Parameter
     - 필터, 정렬, 확장 옵션 등의 "조회" 느낌의 행위를 할 때 쓰면 좋다.
2. DELETE 메서드 만들 떄
  1) Idempotent, "멱등성"
     - 같은 요청을 여러 번 보내더라도 서버 상태는 변하지 않음
       > 주의! "항상 같은 응답을 준다 ← X"
3. 메서드 설계 순서
  - 정답은 아니지만, 생성, 삭제, 수정 메서드부터 하는게 좋다.
    > 프로그램의 뼈대를 잡아두는 역할이고, GET은 어짜피 읽어오는 것이니까.
4. 게시판의 "좋아요" 기능에 대한 고찰
  - 본질은 "상태 변경"이 아니라, "존재 / 비존재"이다.
  - PATCH로 뭔가를 수정할 수 있어야하는데, 수정할 게 없다. "있고 없고밖에 없다."
    > Content-Type 정의가 필요없다.
