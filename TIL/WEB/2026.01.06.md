## GraphQL(GQL) 이란?
 - 페이스북에서 만든 쿼리언어. "웹 클라이언트"가 서버로부터 데이터를 효율적으로 가져오기 위해 쓰는 쿼리 언어.
 - 비교) SQL : DB에 저장된 데이터를 효율적으로 가져오기 위해 쓰는 쿼리 언어
 - DBMS, 네트워크 방식에 종속적이지 않음.
 - REST API와 비교
   1) REST API : URL, METHOD 등을 조합해서 사용함에 따른 다양한 Endpoint
                     각 Endpoint마다 데이터베이스 SQL 쿼리가 달라진다.
   2) GQL API : 단 하나의 Endpoint
                     GQL 스키마 타입마다 데이터베이스 SQL 쿼리가 달라진다.
 - 구조
    "쿼리(Query, 데이터를 읽어옴 - C)" / "뮤테이션(Mutation, 데이터를 변조함 - RUD)" 형태

## 웹 기초 복습 - 놓쳤던 부분 재정리
1. DNS 레코드
  - 부가 정보를 적는다고 이해하자.
    A : 주소, IPv4를 적는다. CNAME : 별명, 도메인의 또다른 이름을 지어준다.

2. curl 명령어
  - curl -v "URL" : verbose 모드, 요청/응답 헤더를 상세하게 출력
  - curl -X GET/POST/PUT/PATCH/DELETE "URL" : HTTP 메서드 사용
  - curl -i "URL" : 응답 헤더 출력

3. HTTP Response / Request
  - header에 구성 요소들은 IANA에 이미 규칙처럼 정해져있다. 그걸 따르면 된다.
  - 접두사 "X"를 붙여서 원하는 이름으로 커스텀 헤더를 구현할 수 있다.

4. Python - requests 주요 명령어 정리
  - 응답받은 response에 대해서 자주 쓰는 함수
 ``` 
 # 내가 보낸 request 객체에 접근 가능
  response.request

 # 응답 코드
 response.status_code

 # 200 (OK 코드)이 아닌 경우 에러 raise
 response.raise_for_status()

 # json 속성을 통해 딕셔너리 타입으로 바로 변환
 response.json()

 # text 속성을 통해 UTF-8로 인코딩된 문자열을 받을 수 있다.
 response.text

 # content 속성을 통해 바이너리 타입으로 데이터를 받을 수 있다.
 response.content

 # encoding 정보 확인
 response.encoding
 ```
