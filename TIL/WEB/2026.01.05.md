## 📝 배운 내용 (Today I Learned)

### 1. 클라이언트-서버(Client-Server) 모델
#### "역할을 분리" 하는 프로그래밍 설계 방식, 요청~응답 형태로 이루어진다.
- 클라이언트 : 서비스를 "요청(Request)"하는 단 
- 서버 : 서비스를 "제공", "응답(Response)"하는 단
  1) 장점 : 중앙 집중 관리를 통한 제어 용이, 보안성 강화, 확장성, 유연성
  2) 단점 : 단일장애점(SPOF), 네트워크 의존(필수로 연결되어야 함), 클라이언트 증가 시 부하 발생, 서버 구축 & 유지보수 등 제반 비용 발생
     > 단일장애점(Single Point Of Failure) : 한 부분에서 문제되면 서버 전체가 다 망가지는 현상

### 2. DNS (Domain Name System)
#### 인터넷 상의 "전화번호부"
- 구조)
```
http://   www   .   naver   .   com   . #우측 부터 단계적으로 접근한다.
    sub-domain / domain name / Top-Level domain / Root domain
```
- DNS 레코드 : DNS를 어떤 방식으로 저장할 지를 가리키는 것, AAAA(IPv6), MX, TXT, NS 등이 있지만 "A", "CNAME"을 기억하자.
  > A(Address) : IPv4, CNAME(Canonical Name) : 별칭을 지정해주는 방식
```
dig naver.com # DNS 설정 조회하는 명령어.
```

### 5. HTTP (HyperText Transfer Protocol)
#### 데이터를 주고 받기 위해 정해둔 일종의 규약
- 주요 2원칙
  1) Connectionless(비연결성) : 한번 요청 또는 응답이 끝나면 그 연결을 바로 끊어진다.
  2) Stateless(무상태성) : 이전에 요청한 것을 기억하지 않는다.
- HTTP Request(요청) / Response(응답)
  1) Request : CRUD 작업을 진행하기 위해 메서드 형태로 요청한다. (GET, POST, PUT, PATCH, DELETE)
     - 요청 시 **가장 첫번째 줄에 정의**
       1) GET : 읽어오기(조회), 몇번을 실행해도 같은 결과가 나온다. **"멱등성(Idempotent)"**
       2) POST : 게시하기(생성), 새로운 데이터를 생성.
       3) PUT : 놓기(수정), 데이터를 대체.
       4) PATCH : 수선하기(일부수정), 데이터 일부 수정.
          > GraphQL 에 대해 공부 필요.
  3) Response : 요청에 대해 응답한다. 이슈 발생 시 상태 코드와 함께 리턴.
     - 1xx) 처리중 / 2xx) 성공 / 3xx) 리다이렉션 / 4xx) 클라이언트 에러 / 5xx) 서버 에러

### 6. REST API (REpresentation State Transfer Application Programming Interface)
- REST : "상태의 표현을 전송"
- API : 각각의 프로그램 간 데이터를 주고 받을 때 또는 실행시킬 때 준수해야 하는 **일관된 규칙**
  > 프로그램 개발 시, REST API docs 문서 작성을 잘 하는 것 또한 역량.
- REST API 설계 원칙
  **1) 명사를 사용하자
  2) 복수형을 사용하자
  3) 계층 구조를 표현하자
  4) 소문자, 하이픈(-)을 활용하자
  5) 확장자를 포함하지 말자**
  > 마치 명사 ~ 동사 처럼, "자원과 행위를 분리"한다고 생각하자. 대상~동작을 떠올려라.

### 7. URL (Uniform Resource Locator)
- 자원의 위치를 일관되게 저장한 것
- 구조)
  ```
  https://  api.example.com  :   443   /   user/jm?   user=admin    #profile
  scheme |     Host          |  Port   |   Path   |   Query       | Fragment
  ```
 
### 8. JSON (JavaScript Object Notation)
- 자바스크립트 기반으로, 사람이 읽기 쉬우면서 컴퓨터가 빠르게 처리할 수 있는 텍스트 기반 형식
  1) 키 값은 반드시 큰따옴표 안에 넣는다. ("")
  2) 주석은 사용할 수 없다.
  3) 마지막에 쉼표가 없다는 것을 잊지 말자.
     > IDE에서 json 파일 생성 시 , **"Reformat" 기능** 활용하면 쉽게 만들 수 있다.

### 9. 직렬화(Serialization) vs 역직렬화(Deserialization)
