### Zone 기반 방화벽 설정 실습
- VyOS 기준, 각 인터페이스에 물려있는 서브 네트워크에 ***"Zone"***을 설정할 수 있다.
  > 적용 즉시 Deny-All 상태로 변경됨.


1. Zone 정의
   ```
   set zone-policy zone <zone이름. public, private 등등...> interface <해당 서브넷이 연결된 이더넷ID> 
   ```

2. 방화벽 정책 생성
   ```
   set firewall name <정책이름. PRIVATE_TO_PUBLIC 이런 느낌으로...> rule <룰 번호. 임의 지정 가능> action accept

   ※ action은 reject도 있긴하다.
      그렇지만, 기본적으로 방화벽은 Deny All 이기 때문에 Accept로 예외를 만들어준다고 생각하자. 
   ```

3. ***"★Stateful★"*** 하도록 방화벽 설정
   ```
   set firewall name <정책이름> rule <룰번호> state established enable
   set firewall name <정책이름> rule <룰번호> state related enable
   set firewall name <정책이름> rule <룰번호> protocol all

   ※ Established & Related인 트래픽★"만!!!!"★ 허용한다. (그 트래픽에 대해 Protocol에 대한 제한은 없다.)
   ```
   - Established 되어있다 : firewall의 테이블에 등록되었다는 의미, Stateful 하도록 변경됨.
   - Related 되어있다 : FTP 같은 예를 들면, 20번 21번 외에 필요한 포트가 있을텐데 그거까지 고려해서 허용해준다.


4. 기타 포트 및 상세 지정할 방화벽 설정
   ```
   ① 공인 IP 대역 → 사설 대역, 22번 포트를 열어줘라의 예시
   set firewall name PUBLIC_TO_PRIVATE rule 10 action accept
   set firewall name PUBLIC_TO_PRIVATE rule 10 protocol tcp 
   set firewall name PUBLIC_TO_PRIVATE rule 10 destination port 22

   ② 사설 대역 → DMZ 대역, 23번과 80번 포트를 열어줘라의 예시
   set firewall name PRIVATE_TO_DMZ rule 30 action accept 
   set firewall name PRIVATE_TO_DMZ rule 30 protocol tcp 
   set firewall name PRIVATE_TO_DMZ rule 30 destination port 23,80
   set zone-policy zone dmz from private firewall name PRIVATE_TO_DMZ

   ※ 리턴이 와야하기 때문에, 반대 방향 트래픽에 대해서 Stateful 하도록 방화벽 설정해줘야 통신 가능!!!!!!
   ```

5. 설정한 방화벽에 대해 Zone에 할당 및 활성화
   ```
   set zone-policy zone <목적지Zone> from <출발지Zone> firewall name <정책이름>

   ※ 출발지와 목적지를 헷갈리지 말자! 
   ```


> show firewall, show zone-policy로 작성한 정책이랑 Zone 설정이 잘 되었는지 항시 점검하자.
