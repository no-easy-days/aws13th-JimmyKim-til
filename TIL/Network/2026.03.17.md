## 📝 배운 내용 (Today I Learned)

### Switch의 동작 원리
- ***"MAC Address 기반"*** 통신! 스스로 특정 인터페이스 포트에 연결된 장비의 MAC Address를 알아낸다.
  > ARP(Address Resolution Protocol) Flooding을 통해 학습한다!

  
- Switch 명칭에 따른 분류
  - L2 스위치 : 일반적인 스위치. 2 Layer 통신
  - L3 스위치 : "IP"를 알고 있다. 그걸 가지고 일종의 ***"라우팅"*** 을 할 수 있다.
  - L4 스위치 : "Port"를 알고 있다. 그걸 가지고 ***"로드밸런싱"*** 을 할 수 있다. → AWS에서 Network Load Balancer 개념
  - L7 스위치 : "서비스"를 알고 있다. 그걸 가지고 ***"로드밸런싱"*** 을 할 수 있다. → AWS 에서 Application Load Balancer 개념

### VLAN
- Switch의 인터페이스 포트에 특별한 꼬리표(Tag)를 통해 네트워크를 분리한다. 서브네팅은 아니다!!
  - VLAN : Layer ***2***에서 ***MAC 주소를 기반***으로 ***스위치***가 분류.
  - Subnet : Layer ***3***에서 ***IP 주소를 기반***으로 ***라우터***가 분류.
- 그렇지만, 결국 Broadcast를 해서 Flooding을 하고, ARP를 활용해야하기 때문에 보통은 같이 묶어서 쓴다.
  > 인터페이스 설정 들어가서 VLAN 값을 부여할 때, Network주소/Prefix 정보도 넣어주는건 결국 Layer 3에 대한 정보도 같이 주는 것...

  
### OpenVSwitch
- ***"서로 다른 HOST 서버" 위***에 존재하는 가상의 네트워크끼리 통신하기 위한 ***"Overlay-Network"를 구성***할 수 있는 모듈.

#### ▶ VPN : "Private Network" 끼리의 Overlay-Network!!!
- 대상 : 물리적으로 떨어져있는(데이터센터 ~ Availability Zone) Private Network 또는 그에 속하는 End Host

#### ▶OpenVSwitch : Host Server 內의 "Virtual Network" 끼리의 Overlay-Network!!!
- 대상 : Hypervisor 또는 특정 Host 내의 Guest VM이나 ***"컨테이너"***


1. OpenVSwitch 관련 패키지 설치
   ```
   yum -y install epel-release https://www.rdoproject.org/repos/rdo-release.rpm
    → 패키지 설치를 위한 외부 레포지토리 등록.
      ※ RDO(Rpm Distribution of Openstack) : Redhat 기반 Linux에 Openstack 플랫폼을 제공하는 무료 배포판. 여기에 OVS 패키지있음!

   yum -y install wget openssl-devel gcc make python-devel openssl-devel kernel-devel graphviz kernel-debug-devel
          autoconf automake rpm-build redhat-rpm-config libtool python-twisted-core python-zope-interface PyQt4
          desktop-file-utils libcap-ng-devel groff checkpolicy selinux-policy-devel
    → OVS 구동을 위한 의존성 패키지 설치
   
   mkdir -p ~/rpmbuild/SOURCES
    → rpm build할 타겟 디렉토리 생성
   
   wget http://openvswitch.org/releases/openvswitch-2.5.4.tar.gz
    → wget으로 OVS 배포 버전 다운로드

   cp openvswitch-2.5.4.tar.gz ~/rpmbuild/SOURCES/
    → rpm build 디렉토리 준비. 여기에는 소스코드가 저장되는 디렉토리이며, ovs 소스코드를 저장한다.
      ※ RPM을 Build 하기 위해선 정해진 디렉토리 구조가 있다. (BUILD / RPMS / SOURCES / SPECS / SRPMS)
       
   tar xfz openvswitch-2.5.4.tar.gz
    → 소스코드 압축 해제

   rpmbuild -bb --nocheck openvswitch-2.5.4/rhel/openvswitch-fedora.spec ★★
    → RPM 빌드 명령. 이 Spec 파일을 참조하고, 무결성 검증 필요없으며 RPM 패키지를 바이너리형태로 빌드해라. 

   yum localinstall -y ~/rpmbuild/RPMS/x86_64/openvswitch-2.5.4-1.el7.x86_64.rpm
    → 빌드된 RPM을 yum 을 통해서 설치. local에 있어서 localinstall
   ```
