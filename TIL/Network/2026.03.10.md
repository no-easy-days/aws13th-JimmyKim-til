### Site-To-Site VPN
<img width="1095" height="552" alt="image" src="https://github.com/user-attachments/assets/de4d0c23-62d2-41a9-85a2-73d580c56299" />
- 여기서 의미하는 "site" 는 Private Nework(사설 망)이다.

  
- ***"사설망끼리" 연결***을 하고 싶다. Client-Server VPN과 마찬가지로 Tunneling을 활용하는 건 같지만 기법이 달라진다.


- Tunneling을 하기 위해서는 결국 레거시(Legacy) 네트워크 망을 활용해야 하는건 마찬가지이다.


- 터널링을 뚫는 방식 : **"IKE"** (Internet Key Exchange)


- 수립한 가상 사설 네트워크를 암호화 하는 방식 : **"ESP"** (Encapsulating Security Payload)


#### Legacy Network = "Underlay Network", Tunneling 적용된 Network = "★★★Overlay★★★ Network"!!!
※ Overlay ~ Underlay의 개념은 가용영역(AZ) 끼리 통신이 되게 만드는 원리로 생각하면 이해가 빠를듯.


1. 외부 접점 인터페이스에 대하여 IPsec 기반 vpn 정의
   ```
   set vpn ipsec ipsec-interfaces inerface eth0

   ※ 모든 세팅의 출발점이다. 외부 네트워크와 통신이 되는 인터페이스를 지정.
   ```
2. esp 기반 암호화 방식이 적용된 그룹 정의 및 생성
   ```
   ① set vpn ipsec esp-group <원하는 esp그룹이름> compression disable
     - esp 방식 암호화를 쓰는 그룹을 하나 선언한다. 압축은 하지 않는다.

   ② set vpn ipsec esp-group <원하는 esp그룹이름> lifetime <원하는 정책 유지 시간(초)>
     - 정책 유지 시간 설정.

   ③ set vpn ipsec esp-group <원하는 esp그룹이름> mode tunnel
     - 해당 esp 그룹에 터널링을 적용할 것이라고 선언.

   ④ set vpn ipsec esp-group <원하는 esp그룹이름> pfs enable
     - PFS(Perfect Forward Secrecy)를 활성화한다. 매 세션마다 새로운 키를 부여한다는 의미
   ```
3. 정의한 ESP 그룹에 대해 암호화/해시 알고리즘 후보 세트(=proposal) 설정
   ```
   ① set vpn ipsec esp-group <원하는 esp그룹이름> proposal 1 encryption <암호화알고리즘>
     - 인증 시 요구되는 암호화 알고리즘 적용. (양방향이 필요하다. AES256이 무난할듯)

   ② set vpn ipsec esp-group <원하는 esp그룹이름> proposal 1 hash <해시알고리즘>
     - 무결성 검증을 위한 해시 알고리즘 적용한다. (제일 무난한 SHA256)

   ※ Proposal 숫자가 낮은게 더 우선순위가 높다. Peer에게 나 이런 알고리즘써~ 느낌으로 "제안"하는 것.
   ```
4. IKE 기반 터널링 방식이 적용된 그룹 정의 및 생성
   ```
   ① set vpn ipsec ike-group <원하는 ike그룹이름> key-exchange ikev1
     - 키값 교환 시 ike Version 1 방식을 쓰겠다.
   ※ 실무에서는 ikev2 를 더 많이 쓴다고 하네. 일단 실습이니까.
   
   ② set vpn ipsec ike-group <원하는 ike그룹이름> ikev2-reauth no
     - ike Version 2 를 쓸 때 재인증 프로세스 미적용하겠다. (ikev2를 안써서 의미는 딱히 없긴하다)
 
   ③ set vpn ipsec ike-group <원하는 ike그룹이름> lifetime <원하는 정책 유지 시간(초)>
     - 정책 유지 기간 설정.
   ```
5. 정의한 IKE 그룹에 대해 암호화/해시 알고리즘 설정
   ```
   ① set vpn ipsec ike-group <원하는 ike그룹이름> proposal 1 encryption <암호화알고리즘>
     - 인증 시 요구되는 암호화 알고리즘 적용. (양방향이 필요하다. AES256이 무난할듯)

   ② set vpn ipsec ike-group <원하는 ike그룹이름> proposal 1 hash <해시알고리즘>
     - 무결성 검증을 위한 해시 알고리즘 적용한다. (제일 무난한 SHA256)
   ```
6. 실질적인 site-to-site 방식 VPN 설정 시작
   ```
   ① set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> authentication mode pre-shared-secret
     - VPN에서 Peer를 맺을 site의 인터페이스 할당하고, 사전 공유 키 방식으로 인증함

   ② set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> authentication pre-shared secret <원하는 사전공유키값>
     - 사전 공유 키값 지정.

   ③ set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> ike-group <원하는 ike그룹이름>
     - 위에서 정의했던 IKE 그룹 설정을 할당한다.

   ④ set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> esp-group <원하는 esp그룹이름>
     - 위에서 정의했던 ESP 그룹 설정을 할당한다.

   ⑤ set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> local-address <외부 네트워크 접접 인터페이스 IP주소>
     - VPN으로 이어줄 나의(Local) 외부 인터페이스 IP 주소 지정

   ⑥ set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> tunnel 0 allow-nat-networks disable
     - NAT 처리 되어있는, 주소 변환된 트래픽은 비허용하겠다.

   ⑦ set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> tunnel 0 local prefix <내부대역 & Prefix>
     - VPN으로 이어주려는 나의 내부 네트워크 & 프리픽스를 정의

   ⑧ set vpn ipsec site-to-site peer <타겟 인터페이스 IP주소> tunnel 0 remote prefix <내부대역 & Prefix>
     - VPN으로 이어주려는 타겟의 네트워크 & 프리픽스를 정의 
   ```
7. VPN 적용 결과 확인
  - run show vpn ipsec sa
    <img width="827" height="173" alt="image" src="https://github.com/user-attachments/assets/60a555f6-6a08-4802-a624-722846ec71dd" />


  - sa : Security Association, ***"보안 세션"*** 을 의미하며, 두 VPN 장비가 어떤 암호 알고리즘, 키로 트래픽을 보호할지 정의해둔 집합체.


※ 구성 및 VPN 연결이 완료하고 난 뒤에, PAT 설정을 추가로 하게되면 VPN 에러가 발생한다.
  - PAT로 나가는 트래픽이랑, ***다른 Site(타겟 사설 망)와 통신하는 트래픽이 PAT가 되지않도록*** 구분해서 설정해야 한다!

  ```
  ① set nat source rule <PAT 설정된 룰번호보다 낮은 숫자로 세팅> exclude
     - 예외 처리용 Rule을 추가 

  ② set nat source rule <룰번호> source address <내가 가진 내부 대역>
  ③ set nat source rule <룰번호> destination address <타겟 내부 대역>

  ※ Exclude의 원리를 까먹지말기.
  ```
