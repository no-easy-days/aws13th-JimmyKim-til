## 📝 배운 내용 (Today I Learned)


### VPN 개요
- Virtual Private Network. 가상 사설 네트워크.
<img width="837" height="512" alt="image" src="https://github.com/user-attachments/assets/8b64c578-d169-4061-88c6-679b59e0f8f6" />


- 마치 내가 ***"다른 사설 네트워크에 존재하는 것 처럼" 여겨질 수 있도록*** 하는 메커니즘.
- 원래는 ISP 통해서 공인IP를 받는게 정상적인 루트이지만, 일종의 ***"땅굴"*** 을 뚫었다고 생각하자.
  > 이를 터널링(Tunneling)이라고 부른다.


### Server-Client VPN
- 클라이언트가 마치 서버가 속한 네트워크 망 안에 속해있는 것처럼 만들어 주는 기법.
- 그치만 결국, 터널링 작업을 하는 것도 Legacy 네트워크 망을 활용은 하긴 해야하는 걸 잊지말자.
  (밖으로 나가는 방법은 이러나저러나 하나 뿐이다...)
- 터널링을 뚫는 방식 : ***L2TP(Layer 2 Tunneling Protocol)***
- 수립한 가상 사설 네트워크를 암호화 하는 방식 : ***IPsec***


1. 외부 접점 인터페이스에 대하여 IPsec 기반 vpn 정의
   ```
   set vpn ipsec ipsec-interfaces interface eth0
   ```
2. 사설 망에도 VPN 구성을 허용하도록 설정
   ```
   set vpn ipsec nat-traversal enable

   ※ 원래 IPsec은 기본적으로 공인 IP 대역대끼리 구성하는데 그것을 예외 처리해준다는 느낌.
   ```
3. NAT-Traversal을 허용할 대역대 지정
   ```
   set vpn nat-networks allowed-network 0.0.0.0/0

   ※ 모든 대역대를 허용한다.
   ```
4. l2tp 터널링 방식 기반 vpn을 통해 내부로 진입한 Client에 대해 IP 주소 부여 설정
   ```
   set vpn l2tp remote-access client-ip-pool start <부여할 대역대의 첫 IP주소>
   set vpn l2tp remote-access client-ip-pool stop <부여할 대역대의 마지막 IP주소>

   ※ Private 네트워크 망에 "속하는 IP"를 부여하고 물리적으로 연결하겠다는 의미!!
   ```
5. l2tp 터널링 방식 기반 vpn에 대한 접속 방식 설정
   ```
   set vpn l2tp remote-access authentication mode local
   set vpn l2tp remote-access authentication local-users username <ID> password <패스워드>

   ※ 식별은 로컬 방식(ID/PW로 하겠다는 의미)
   ```
6. l2tp 터널링 방식 기반 vpn으로 접속한 대상을 인증하는 방식 설정
   ```
   set vpn 12tp remote-access ipsec-settings authentication mode pre-shared-secret
   set vpn l2tp remote-access ipsec-settings authentication pre-shared secret <사전 공유키로 지정할 값>
   
   ※ 암호화 방식이기 떄문에 ipsec 세팅에서 잡아준다.
   ※ pre-shared-secret : 사전 공유키 라고 칭하며, 이건 방식에 따라 바뀌는거임.
   ```
7. 설정 마친 vpn을 적용할 외부 인터페이스 설정
   ```
   set vpn l2tp remote-access outside-address <이더넷 인터페이스 IP>

   ※ 그 대역에 들어가기 위한 인터페이스의 주소. 클라이언트 입장에서는 "Gateway"가 될 것이다!!!
   ```
