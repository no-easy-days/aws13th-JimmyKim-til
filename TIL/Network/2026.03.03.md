### SSH - secure shell
- 원격 접속 프로토콜. 암/복호화 처리된 메시지를 송수신한다. TCP 22번 포트 사용.
  ※ Telnet : 평문 형태의 메시지 송수신 프로토콜. TCP 23번 포트 사용
- 인증 방식
  1. ID & PW
  2. 키 페어 → 공개 키 & 개인 키로 구성되어 있으며, 비대칭 형태!
     - Private Key(개인 키) : 클라이언트 단 보유, ***"열쇠"***
     - Public Key(공개 키) : 서버 단 보유, ***"자물쇠"***
  3. 암호화
     - 메시지를 어떻게 암/복호화할 지를 정의한다. AES 대칭키 알고리즘이 일반적이고, ***세션을 맺을 때 활용***.

### ACL - access list
- 접근 제어 리스트
#### 1. 표준(Standard) ACL
- ***"1~99 & 1000~1999"의 리스트 ID 범위***를 가진다.
- 출발지 주소(Source IP Address)를 제어하며, 위에서 아래로 순차적으로 읽어간다.
- "우선 정의된 ACL 범위에 해당하면 밑의 내용은 폐기"된다. → ***위쪽에 먼저 충족된 ACL이 있을 경우 그 규칙을 바로 따라간다!***
- 항상 "Deny ALL" 이 생략되어 있다. 이것을 우리는 ***"암묵적 거부"*** 라고 칭한다.
- 라우터를 기준으로 인터페이스의 IN 혹은 OUT 마다 한개의 ACL만 적용 가능하다. ***"라우터의 입장에서 유입/유출을 구분지어 생각"하자!***
