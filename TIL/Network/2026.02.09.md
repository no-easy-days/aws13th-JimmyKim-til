### OSI 7 Layer
> Open Systems Inter(외부! ↔ Intra : 내부)-Connection 7 Layer. 시스템 끼리 통신을 할때 지켜야하는 프로토콜을 7계층으로 분리한 것.

<img width="1230" height="680" alt="image" src="https://github.com/user-attachments/assets/bd1d50f6-09e6-45ad-ba50-bebc6ec865e8" />

- SDU(Service Data Unit) : 상위 계층에서 내려온 블록. 하위 계층으로 갈수록 SDU에 계속해서 Header 느낌으로 정보를 붙여나간다.
- PDU(Protocol Data Unit) : Decapsulation 또는 Encapsulation(계층 간 이동) 진행 시 주고 받는 블록.

1. 4계층(Transport Layer) : 전송방식과 포트번호를 정의하고 결정한다. 
   - TCP(Transmission Control Protocol) : **신뢰성 있는** 통신 방식. "정확도"(3-Way Handshake)가 생명이다. (HTTP, HTTPs, SSH, FTP 등)
   - UDP(User Datagram Protocol) : **신뢰성 없는** 통신 방식. "흐름"이 중요하다. (DHCP, tFTP 등)

2. ***3계층(Network Layer)*** : IP(**32비트 주소체계!!!**)를 정의하고 결정한다. ***장비 : "라우터"***
   - Header에 Source IP / Destination IP 정보가 담겨 있다는 것을 잊지 말자.
  
3. 2계층(Data-Link Layer) : MAC 주소를 정의하고 결정한다. ***장비 : 스위치***
   - Header에 Source MAC Address / Destination MAC Address 정보가 담겨있다는 것을 잊지 말자.

#### 각 계층이 만나는 Interface 지점에서는 Encapsulation/Decapsulation이 계속 발생하며, 그 계층의 장비가 알 수 있는 단계까지 바꿔줘야함.
> N 계층의 장비는 N계층 "이하"의 헤더 정보를 열어 볼 수 있지만, N+1 이상의 헤더 정보는 식별할 수 없다.


### Classful Network

1. A Class : Network ID가 8Bit / Host ID가 24Bit, ***맨 앞 Bit가 0*** 으로 고정.
   - 10.0.0.0 ~ 10.255.255.255
3. B Class : Network ID가 16Bit / Host ID가 16Bit, ***맨 앞 Bit가 10*** 으로 고정.
   - 172.16.0.0 ~ 172.31.255.255
5. C Class : Network ID가 24Bit / Host ID가 8Bit, ***맨 앞 Bit가 110*** 으로 고정.
   - 192.168.0.0 ~ 192.168.255.255
7. D Class : 연구 목적
8. E Class : 특수 목적
   > Host ID를 바탕으로 그 네트워크의 규모를 알 수 있다. 각 Class 별 Private 네트워크 대역을 벗어나면 그것은 공인 IP일 것이다.


### Subnet Mask

- 네트워크의 규모를 결정한다. *어디까지 Network ID / Host ID 인지*를 식별하는데 쓰인다.
- ***1 : Fix / 0 : Don't Care***. 1은 항상 연속되어야 하며, 0 다음 1이 올 수 없음.
  > 각 Class는 반드시 서브넷 마스크의 규칙을 준수해야 함. 그렇지만 비효율적이라 요즘에는 Classless 하게 네트워크를 구성함.


#### 네트워크 표현 시, 가장 작은 주소(Network ID 만을 기입, 이것이 대표 주소)를 적고, 서브넷 마스크 또는 Prefix로 기입해서 표현한다.
- 결론 : 모든 서브넷들이 사용할 수 있는 호스트 수(가용 IP 수) = 네트워크 규모 - 2

#### 꿀팁 : Host ID 부분을 0으로 채우면 그 것이 가장 작은 주소이자 대표 주소 / 1로 채우면 그 것은 Broadcast 주소.
