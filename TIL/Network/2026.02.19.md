### Linux 설치하면 좋은 패키지들
1. wget : Web페이지에서 http get 으로 파일 다운로드를 할 수 있도록 해주는 패키지
2. unzip : 압축 파일 관리 패키지
3. tcpdump : 수신 트래픽 확인 모듈이 포함된 패키지.
   ※ 사용방법) tcpdump -i <인터페이스이름> icmp : Ping 명령은 icmp를 활용하기 때문에, 핑 때려놓고 이 명령어로 추적한다.


### DHCP Server
- Cisco 때와 개념은 완전히 동일하며, 기본적으로 자신의 인터페이스가 속한 네트워크에 IP를 부여한다.
- 패키지 설치가 필요하다. yum install -y dhcpd
- 설치 후, vi /etc/dhcp/dhcpd.conf 를 통하여 수정하며, 수정 완료 후 systemctl로 재시작 해줘야한다.
  <img width="1088" height="527" alt="image" src="https://github.com/user-attachments/assets/f4ad272e-ea8d-4e52-ba20-fb4d103c6324" />


- 클라이언트는 vi /etc/sysconfig/network-scripts/<이더넷 인터페이스명>에서 BOOTPROTO="yes" 또는 "dhcp"로 변경해야 IP를 받아올 수 있다!

  
  <img width="822" height="427" alt="image" src="https://github.com/user-attachments/assets/64957156-7a7e-4a5f-939d-601e10e2bb2b" />


#### ※ Helper-Address
  - 클라이언트가 DHCP-Discover는 항상 본인이 속해있는, 같은 네트워크 망에만 요청한다.
  - 그렇기 때문에, 다른 서브넷에 뿌려주려면 Router에서 ***"Helper Address(여기는 DHCP 서버의 주소를 입력)"*** 를 지정해줘야한다!!
  - 그러면, 다른 서브넷에서도 DHCP-Discover 메시지가 도달 가능하다.
    > Cisco에서 ip helper-address <DHCP서버주소> 로 지정한다.
