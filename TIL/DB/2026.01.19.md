1. 데몬(Daemon)
 - 사용자가 직접적으로 제어하지 않고, "백그라운드"에서 돌면서 여러 작업을 수행
 - 유닉스/리눅스 계열, 프로그램뒤에 "d"가 붙어있는 것이 규칙


2. 관계형 데이터베이스(RDBMS)
 - "상호 연관"된 데이터를 "테이블 형태"로 저장하고 로우와 컬럼 형태로 저장 & 제공하는 저장소


3. 스키마(Schema) (스키마 > 테이블 > 데이터)
 - 테이블, 인덱스, 뷰, 프로시저, 트리거 등등 모든 것들을 포함하는 대단위.
 - MariaDB or MySQL에서는 Schema와 Database는 동일한 의미로 사용 (PostgreSQL은 포함 관계 더 있음)
   1) 개념적 스키마
     - 손으로 슥슥 그려서 구조를 나타내는 것. 
     - "시스템 이해", "커뮤니케이션", "문서화"
   2) 논리적 스키마
     - 개념적 스키마를 기반으로 보다 구체적으로 표현. "테이블", "컬럼", "데이터유형", "제약 조건" → ERD
     - "데이터 무결성", "독립성", "구조적 설계 가능"
   3) 물리적 스키마
     - 논리적 스키마를 기반으로 실제로 만들어 냄. "쿼리"를 통해 수행. "인덱스", "파티셔닝"
     - "성능 최적화", "구현 세부사항 설계", "자원 관리"


4. 테이블
 - 데이터베이스에서 데이터를 구조적으로 저장하는 기본 단위.
 - "구조화된 데이터 저장", "효율적인 데이터 검색", " 관계 설정 가능"
 - row, column으로 구성, row는 "레코드", column은 "어트리뷰트"라고도 부름.
   1) 기본 키(Primary Key, PK) : 각 레코드를 고유하게 식별하는 하나의 컬럼 or 여러 컬럼의 조합.
     - 유일해야하고, NOT NULL이며, 불변해야 한다! 
   2) 외래 키(Foreign Key, FK) : 다른 테이블의 PK를 참조하는 컬럼


5. 인덱스
 - 데이터베이스에서 데이터 검색 속도를 향상시키기 위해 사용. 특정 컬럼을 대상으로 생성한다.
 - 대용량 데이터에서 특정 데이터를 빠르게 찾아야할 때 유용하게 사용. CREATE INDEX로 생성. (수정 불가)


6. 쿼리
  a) DDL : Data Definition Language, 테이블 구조 정의
  b) DML : Data Manipulation Language, 데이터 조작
  c) DCL : Data Control Language, 권한 관리
 - 하고 싶은 행위를 주석으로 한번 써보고 작성해보면 실수 할 일이 없다.
 - Multi-line Insert : 한번의 Insert 문으로 여러 행의 데이터를 동시 삽입, 개별 여러번 보다 성능적으로 효율적
 - SELECT 쿼리 실행 순서 : FROM→JOIN→WHERE→GROUP BY→HAVING→SELECT→DISTINCT→ORDER BY→LIMIT
 - Soft Delete 데이터를 다른 테이블으로 옮기거나 감추는 행위 / Hard Delete : 데이터를 완전히 지워버리는 행위
 - DELETE 쿼리 중 FK 에러 발생 시 : 테이블 참조하는 곳 확인하여, 그 테이블 "부터" 먼저 삭제 해야한다.


7. JOIN
 - 데이터베이스에서 여러 테이블의 데이터를 결합하여 하나의 결과 형태로 만드는 것.
 a) INNER JOIN : 두 테이블에서 일치하는 데이터만 가져오기. ON 절에 연결 조건 작성
 b) LEFT JOIN : "FROM 뒤에 테이블이 왼쪽", 왼쪽 테이블의 모든 행을 결과에 포함. 
 > 수학의 집합을 연관지어 생각하면 이해가 쉽다!


### Cursor 이해하기
  > 데이터베이스에서 특정 행을 가리킨다.
 - 커서가 뭔가 실행을 하면, fetchall 을 통해 결과를 가져와야하고, 다쓰면 close까지!
 - charset = 'utf8mb4'를 사용하는 이유 : "이모지를 포함"한 모든 유니코드 문자 지원.
 - fetch : 실제로 데이터를 꺼내오는 동작. "가져오다"
 - cursorclass = DictCursor : 딕셔너리 처럼 값을 가져올 수 있음. 직관적이어서 코드 이해가 빨라진다.
